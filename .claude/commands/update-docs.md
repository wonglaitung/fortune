# 更新文档 SOP

每次功能更新后，按以下顺序更新文档并提交 GitHub。

## 步骤 1：更新进度文件

更新 `progress.txt`：

1. 在"最新进展"部分添加本次更新的条目
2. 包含：更新内容、核心发现/方案、验证效果（如有）、改动文件
3. 使用表格展示关键数据
4. 旧进展自动下移，保持最新进展在最上方

## 步骤 2：更新经验教训

更新 `lessons.md`：

1. 在相应章节添加新的经验教训（按主题分类）
2. 格式：问题 → 现象 → 解决方案 → 教训
3. 更新版本号和更新日志

## 步骤 3：更新 CLAUDE.md

从项目整体角度更新，**不要**强调"最新修改"：

1. **核心警告**：添加新的关键警告（如有）
2. **数据流架构**：更新架构图（如有重大变更）
3. **机器学习模型**：更新模型性能数据（使用验证过的最新数据）
4. **常用命令**：添加新命令（如有）
5. 保持当前颗粒度，删除过时内容

## 步骤 4：更新 README.md

从项目整体角度更新，**不要**记录项目状态：

1. **项目简介**：功能概述
2. **核心功能**：列出主要功能模块
3. **快速开始**：安装和运行命令
4. **模型性能**：使用验证过的最新数据
5. 保持当前颗粒度

## 步骤 5：全面对比 README.md

**重要**：更新后必须全面对比 README.md 的内容，检查是否反映项目最新情况。

检查清单：

1. **恒指三周期预测准确率**：是否与最新验证结果一致？
   - 1天、5天、20天准确率
   - 八大交易模式准确率（假突破101、下跌中继001等）

2. **个股 Walk-forward 验证结果**：是否与最新验证结果一致？
   - 平均准确率、夏普比率、IC、Rank IC
   - 验证日期是否更新

3. **特征重要性 Top 10**：是否与最新验证结果一致？
   - 特征名称、重要性数值
   - 验证日期是否更新

4. **核心警告**：数据是否准确？
   - 恒指 vs 个股准确率对比

5. **验证日期**：所有验证日期是否更新为最新？

## 步骤 6：提交 GitHub

```bash
# 检查修改
git status
git diff

# 添加文件（只添加 .md 文件，不添加 .json/.csv）
git add *.md docs/*.md lessons.md progress.txt CLAUDE.md README.md

# 提交
git commit -m "docs: 更新文档 - <简短描述>"

# 推送
git push
```

## 注意事项

- 使用检查过的最新验证数据，不要使用过时数据
- 文档从项目整体角度描述，不强调"最新修改了什么"
- 保持文档当前颗粒度，不要过度细化或过度简化
- 只提交 `.md` 文件，不提交 `.json`/`.csv` 等数据文件
