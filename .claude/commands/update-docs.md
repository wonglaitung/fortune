# 更新文档 SOP

每次功能更新后，按以下顺序更新文档并提交 GitHub。

## 步骤 1：更新进度文件

更新 `progress.txt`：

1. 在"最新进展"部分添加本次更新的条目
2. 包含：更新内容、核心发现/方案、验证效果（如有）、改动文件
3. 使用表格展示关键数据
4. 旧进展自动下移，保持最新进展在最上方

## 步骤 2：更新经验教训

更新 `lessons.md`：

1. 在相应章节添加新的经验教训（按主题分类）
2. 格式：问题 → 现象 → 解决方案 → 教训
3. 更新版本号和更新日志

## 步骤 3：更新 CLAUDE.md

从项目整体角度更新，**不要**强调"最新修改"：

1. **核心警告**：添加新的关键警告（如有）
2. **数据流架构**：更新架构图（如有重大变更）
3. **机器学习模型**：更新模型性能数据（使用验证过的最新数据）
4. **常用命令**：添加新命令（如有）
5. 保持当前颗粒度，删除过时内容

## 步骤 4：更新 README.md

从项目整体角度更新，**不要**记录项目状态：

1. **项目简介**：功能概述
2. **核心功能**：列出主要功能模块
3. **快速开始**：安装和运行命令
4. **模型性能**：使用验证过的最新数据
5. 保持当前颗粒度

## 步骤 5：遂行校对 README.md

**重要**：更新后必须遂行校对 README.md 的内容，确保反映项目当前真实情况。

### 5.1 数据一致性校对

逐项核对以下数据，与验证文件和 CLAUDE.md 交叉对比：

| 检查项 | 数据来源 | 核对要点 |
|--------|----------|----------|
| 恒指三周期准确率 | CLAUDE.md, THREE_HORIZON_ANALYSIS.md | 1d/5d/20d 准确率是否一致 |
| 恒指八大模式胜率 | THREE_HORIZON_ANALYSIS.md | 假突破(101)、一致看涨(111)、下跌中继(001) 等 |
| 个股 Walk-forward 结果 | validation_summary.json, CLAUDE.md | 准确率、IC、Rank IC、夏普比率、最大回撤 |
| 特征重要性 Top 10 | fold_metrics_detail.json | 特征名称、重要性数值 |
| 验证日期 | 各验证文件 | 是否为最新验证日期 |
| 核心警告数据 | CLAUDE.md | 恒指 vs 个股准确率对比 |

### 5.2 常见错误检查

| 错误类型 | 示例 | 修正方法 |
|----------|------|----------|
| 策略胜率描述错误 | "三周期一致做空（90%胜率）" 实际是 79.80% | 改为实际数据"假突破做多（87%胜率）" |
| 过时数据未清理 | 保留旧版验证的详细 Fold 表格 | 删除过时数据，保持简洁 |
| 日期未更新 | 验证日期停留在旧版本 | 更新为最新验证日期 |
| 数据来源混淆 | 训练时 CV 准确率误作 Walk-forward 准确率 | 明确区分数据来源 |

### 5.3 校对流程

```bash
# 1. 读取 README.md 中的关键数据
grep -E "准确率|胜率|IC|夏普" README.md

# 2. 与验证文件对比
# 恒指数据
grep "81\|65\|51" docs/THREE_HORIZON_ANALYSIS.md | head -5
# 个股数据
cat output/*/validation_summary.json | grep -E "accuracy|sharpe|ic"

# 3. 确认特征重要性
python3 -c "import json; d=json.load(open('output/*/fold_metrics_detail.json')); print(d['folds'][0]['top_features'][:10])"
```

### 5.4 校对确认清单

校对完成后，确认以下各项：

- [ ] 恒指三周期准确率与 CLAUDE.md 一致
- [ ] 恒指八大模式胜率与 THREE_HORIZON_ANALYSIS.md 一致
- [ ] 个股 Walk-forward 结果与 validation_summary.json 一致
- [ ] 特征重要性 Top 10 与 fold_metrics_detail.json 一致
- [ ] 所有验证日期已更新为最新
- [ ] 无过时数据残留（如旧版 Fold 详细表格）
- [ ] 策略描述使用实际胜率，非夸大数字

## 步骤 6：提交 GitHub

```bash
# 检查修改
git status
git diff

# 添加文件（只添加 .md 文件，不添加 .json/.csv）
git add *.md docs/*.md lessons.md progress.txt CLAUDE.md README.md

# 提交
git commit -m "docs: 更新文档 - <简短描述>"

# 推送
git push
```

## 注意事项

- 使用检查过的最新验证数据，不要使用过时数据
- 文档从项目整体角度描述，不强调"最新修改了什么"
- 保持文档当前颗粒度，不要过度细化或过度简化
- 只提交 `.md` 文件，不提交 `.json`/`.csv` 等数据文件
- **README.md 必须遂行校对**：逐项核对数据，确保反映项目当前真实情况
- 策略胜率描述必须使用实际验证数据，禁止夸大或使用过时数据
