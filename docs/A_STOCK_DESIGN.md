## 系统定位

实践**人机混合智能**理念，构建A股智能投资决策辅助系统。系统整合**机器学习定量预测**与**大模型定性分析**，为投资者提供全面、客观、可操作的投资建议。

### 三层决策框架

```
┌─────────────────────────────────────────────────────────────┐
│                    人机混合智能决策框架                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   第一层：机器学习定量（CatBoost）                             │
│   ─────────────────────────────                              │
│   • 三周期概率预测（1d/5d/20d）                               │
│   • 1077个特征，数据驱动                                      │
│   • 客观、可量化、可回测                                       │
│   • 输出：涨跌概率、量化建议                                   │

│   ─────────────────────────────                              │
│   • 技术面、基本面、市场环境综合分析                           │
│   • 政策解读、行业趋势、风险提示                               │
│   • 灵活、上下文理解、应对突发事件                             │

```

### 核心设计理念

| 理念 | 说明 |
|------|------|
| **机器学习负责定量** | 基于历史数据和统计规律，给出概率化预测。优点是客观、可量化、可回测；缺点是无法理解市场上下文、政策变化等定性因素。 |
| **大模型负责定性** | 基于技术指标、市场环境、基本面、政策分析，给出综合判断。优点是灵活、能理解上下文；缺点是可能主观、无法精确量化。 |
| **人来决策** | 系统提供量化建议和AI建议两列，投资者综合参考后做最终决策。两列建议可能冲突，此时需投资者根据自身判断做出选择。 |

## 二、股票池与网络设计

**设计原理与技术要点**：

股票池与网络设计是A股智能分析系统的核心基础设施，其技术实现遵循以下原则：

1. **样本效率与预测聚焦的平衡**：4只核心持仓股权重3.0倍，49只扩展股权重1.0倍，模型在分裂节点时优先优化核心股预测准确率，避免"平均主义"导致核心预测被稀释。

2. **网络特征的个股区分能力**：市场级特征（如中证1000收益率、主力资金净流入）对所有股票同值，无法区分个股表现。通过网络交叉特征（`net_constraint_CN_10Y_Yield`），将市场信号与个股网络位置结合，实现"同一市场环境，不同个股响应"的预测效果。

3. **产业链锚定与信息传导捕捉**：四组网络设计围绕核心持仓股构建产业链上下游、竞争对手、地域联动关系，通过PMFG（Planar Maximally Filtered Graph）算法提取相关性网络，捕捉"大哥涨、小弟跟"、"上游提价、下游承压"的跨股票信号。

4. **双指数基准的风险分层**：中证1000提供全市场中小盘Beta信号，创业板指捕捉成长股风险偏好。核心持仓中石药创新（创业板）对创业板指敏感度高于中证1000，运达科技（主板）则主要跟随中证1000风格，实现"一把钥匙开一把锁"的风险分层监控。

## 三、特征工程

**设计原理与技术要点**：

特征工程是机器学习模型预测能力的核心驱动力，A股特征体系在继承港股成熟架构的基础上，针对A股市场特殊性进行了深度定制：

1. **A股特有特征的创新设计**：涨跌停特征捕捉A股特有的价格限制机制，主力资金特征跟踪大资金流向对个股的影响，行为金融因子（凸显性、球队硬币）基于学术界最佳实践，反映A股投资者非理性行为模式。

2. **网络特征与市场特征的交叉架构**：市场级特征（中证1000收益率、主力资金、利率数据）对所有股票同值，必须与网络特征（中心性、结构洞、社区ID）交叉才能区分个股。交叉设计遵循单调性原则：正向×正向用乘法，正向×负向用风险调整（除法），避免逻辑冲突。

3. **数据泄漏防护机制**：所有 `.rolling()` 计算的特征必须使用 `.shift(1)` 避免当日数据泄漏；未来收益标签使用 `.shift(-horizon)` 确保训练时不使用未来信息；涨跌停、主力资金等实时特征在训练时使用滞后数据。

4. **跨市场联动特征的预测价值**：铜期货收益率对化工板块（渤海化学）有显著预测力，原油期货捕捉成本端传导效应，人民币汇率影响出口导向型企业盈利预期。验证结果显示 `Copper_Return_20d` 进入 Top 5 特征重要性。

### 3.2 行为金融因子（业界最佳实践）

**研究来源**：
- 国信证券：《凸显性因子：行为金融学在A股的创新应用》
- 方正证券：《球队硬币因子：基于体育博彩理论的选股因子》

#### 凸显性因子 (STR)

基于 Kahneman-Tversky 前景理论：投资者对极端收益/损失给予过多关注。

```python
# 计算公式
STR_Nd = (Max_Return_Nd - Min_Return_Nd) / Avg_Abs_Return_Nd

# 特征列表
STR_5d          # 5日凸显性
STR_20d         # 20日凸显性
STR_Intensity   # 凸显强度 = STR_5d / STR_20d
STR_Signal      # 凸显信号（综合排名）
```

#### 球队硬币因子

基于体育博彩理论：当市场情绪过度偏向一方时，反向操作有利可图。

```python
# 计算公式
Up_Ratio = Up_Days / Total_Days  # 上涨天数占比
Team_Coin_Sentiment = (Up_Ratio - 0.5) * 2  # 偏离中性程度

# 特征列表
Team_Coin_Sentiment   # 市场情绪偏向 [-1, 1]
Team_Coin_Reversal    # 反转信号
Team_Coin_Intensity   # 情绪强度
```

**实现位置**：`ml_services/feature_engineering/behavioral_factors.py`

### 3.6 特征单调性设计

交叉特征时保持逻辑单调性：

| 交叉类型 | 交叉方式 | 示例 |
|---------|---------|------|
| 正向 × 正向 | 乘法 | 中心性 × 收益率 |
| 负向 × 负向 | 风险放大 | 约束度 × VIX → `-|X| × |Y|` |
| 正向 × 负向 | 风险调整 | 中心性 × VIX → `X / (|Y| + ε)` |

### 3.7 数据泄漏防护

**高风险特征必须使用 `.shift(1)`**：

| 风险点 | 防护措施 |
|---------|---------|
| 所有 `.rolling()` 计算的特征 | 使用 `.shift(1)` 避免当日数据 |
| 涨跌停特征 | 使用前一日收盘价计算 |
| 主力资金特征 | 使用滞后数据 |
| 行为金融因子 | 使用滞后数据 |

```python
# 正确示例
df['STR_5d'] = (df['Max_Return_5d'] - df['Min_Return_5d']) / df['Avg_Abs_Return_5d']
df['STR_5d'] = df['STR_5d'].shift(1)  # 避免数据泄漏
```

## 四、模型训练

**设计原理与技术要点**：

模型训练是将特征转化为预测能力的核心环节，A股模型训练设计重点解决样本不平衡、跨市场涨跌幅差异、数据泄漏检测三大挑战：

1. **样本权重策略的核心逻辑**：53只股票中仅4只是核心持仓，若完全等权训练，模型会花92%精力优化不打算买的扩展股。核心股权重3.0倍后，CatBoost在分裂树节点时，错误预测核心股的惩罚是扩展股的3倍，自动优先寻找契合核心股特征的路径。

2. **标签标准化解决涨跌幅差异**：主板涨跌停限制10%，创业板20%，若直接使用原始收益率作为标签，创业板极端值会主导MSE损失函数。标签标准化（除以滚动波动率）后，不同板块股票的预测误差在同一量级，模型学习更加均衡。

3. **数据泄漏阈值监控**：A股个股预测准确率正常范围为50-60%，若超过65%通常是数据泄漏信号（特征使用了未来信息）。恒指预测准确率可达81%，但个股预测难度显著更高，需警惕过高的验证指标。

4. **模型验证与特征重要性分析**：20天模型 Top 10 特征中，网络×利率交叉特征占前3位，验证了网络特征设计的有效性；跨市场联动特征（铜期货）进入 Top 5，体现了产业链成本传导逻辑；涨跌停特征重要性高，反映A股市场特殊性。

## 五、运行流程

**设计原理与技术要点**：

运行流程设计遵循"新闻-训练-预测-分析-通知"五步闭环，确保每日收市后自动化生成投资建议：

1. **新闻获取优先于训练**：模型训练需要新闻情感特征，因此新闻获取必须作为第一步执行。新闻数据来自东方财富API，包含个股资讯和四维情感评分。

2. **三周期模型训练时机**：每日收市后（15:00 CST）重新训练模型，使用最新1460天历史数据。模型训练会自动计算新闻情感特征（sentiment_ma3/7/14等），若无新闻数据则使用默认值。

3. **预测前缓存清理机制**：特征缓存（`data/feature_cache/`）加速特征计算170倍，但新增特征后必须清除缓存，否则会因特征维度不匹配导致预测失败。`rm -rf data/feature_cache/*.pkl` 是预测前的必要步骤。

4. **大模型建议的增值作用**：CatBoost模型擅长量化预测（概率、方向），但在基本面分析、行业趋势、政策影响等定性判断上能力有限。通义千问大模型补充定性分析，生成投资建议和风险提示，与量化预测形成互补。

5. **综合分析的决策整合**：将CatBoost三周期预测（量化）、通义千问大模型建议（定性）、新闻情感分析、异常检测（风险提示）、板块分析（市场全景）整合为最终买卖建议，实现"机器量化+AI定性"的决策闭环。

| 量化建议 | 基于CatBoost三周期预测的建议（积极买入/逢低买入/观望等），纯量化信号 | 模式映射 |
| 胜率 | 该模式的历史胜率 | 传导模式验证 |
| 筹码阻力 | 上方筹码阻力（低/中/高） | 筹码分布计算 |
| 盈亏比 | 历史盈亏比（基于Walk-forward验证） | 验证结果分析 |
| 期望收益 | 期望收益率 | 盈亏比计算 |
| 风险得分 | 风险控制得分（0-100） | 风险回报分析 |
| 综合得分 | 综合风险回报得分 | 风险得分×50% + 回报得分×50% |
| 风险建议 | 最终风险建议（优选/推荐/观察/暂缓） | 综合得分评级 |

**量化建议 vs AI建议说明**：

| 建议类型 | 来源 | 分析维度 | 特点 |
|---------|------|---------|------|
| **量化建议** | CatBoost三周期预测 | 概率预测（1d/5d/20d） | 纯量化信号，不含定性分析，可能忽略市场情绪、政策变化等因素 |
| **AI建议** | 通义千问大模型 | 技术指标、市场环境、基本面、政策分析 | 定性分析，综合考虑多维度因素，但可能有主观偏差 |

**使用建议**：两列建议可能存在差异（如量化建议"积极买入"但AI建议"观望"），此时应综合判断：
1. 若两者一致，信号更可靠
2. 若量化建议看涨但AI建议谨慎，可能是技术面超买或有潜在风险
3. 若量化建议看跌但AI建议买入，可能是基本面改善或市场环境转好

**AI建议说明文字**：

> **说明**：AI建议基于通义千问大模型，结合技术指标、市场环境、基本面等定性分析，与量化建议（基于CatBoost三周期预测）可能存在差异，请综合参考。

## 七、风险与避坑

**设计原理与技术要点**：

风险防控是量化投资系统的生命线，A股系统重点防范数据泄漏、过拟合、市场结构性差异三大类风险：

1. **时序泄漏的隐蔽性与危害**：网络社区ID如果使用全量历史数据计算，训练前期就会"知道"未来社区结构，导致验证准确率虚高。滚动窗口计算（`window=60`）确保每个时间点只使用过去60天数据，避免前瞻性偏差。类似地，所有 `.rolling()` 计算的特征必须使用 `.shift(1)` 延迟。

2. **交叉特征的过拟合风险**：CatBoost自带非线性组合能力（深度8层），人工设计过多乘法交叉特征可能过拟合。解决方案：保留网络特征和市场特征作为独立特征，让树模型自己学习交叉逻辑；验证时监控训练集与测试集准确率差异，若超过5%通常是过拟合信号。

3. **主板与创业板的结构性差异**：主板涨跌停限制10%，创业板20%，混合训练时创业板极端值会主导MSE损失函数。标签标准化（除以滚动波动率）后，不同板块股票的预测误差在同一量级；训练时也可考虑分层训练（主板模型、创业板模型），但会牺牲样本量。

4. **特征单调性的逻辑约束**：交叉特征时必须保持逻辑单调性。例如"中心性"是正向特征（值越大越利好），"VIX"是负向特征（值越大越利空），直接乘法会导致逻辑冲突（中心性高×VIX高=乘积大，但实际应该是风险调整后的负向信号）。正确做法是用风险调整公式：`X / (|Y| + ε)`。

### 7.1 网络社区ID的时序泄漏

**风险**：用过去3年总数据跑出静态社区ID，在训练前期就知道未来社区结构。

**修正方案**：使用滚动窗口计算社区ID

```python

    for date in df.index:
        lookback_data = df.loc[date - timedelta(days=window):date]
        # 使用 lookback_data 计算相关性矩阵，构建网络
```

### 6.2 交叉特征的"过度共振"

**风险**：CatBoost自带非线性组合能力，人工乘法交叉可能过拟合。

**修正方案**：把网络特征和市场特征作为独立特征，设置 `depth=6`，让树自己学习交叉。

### 6.3 主板与创业板涨跌幅截断

**风险**：模型混合10%涨跌停（主板）与20%涨跌停（创业板），创业板极端值主导MSE。

**修正方案**：标签标准化（除以滚动波动率）

```python
df['label_normalized'] = df['future_return'] / df['volatility_20d']
```

### 7.4 股票代码前导零丢失问题

**风险**：AI返回的股票代码可能缺少前导零（如"2655"而非"002655"），导致查找stock_analyses时失败，显示现价0.00。
