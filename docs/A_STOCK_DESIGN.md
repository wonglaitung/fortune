## 一、系统概述

### 1.1 设计目标

基于港股智能分析系统的成熟架构，构建A股版本，核心能力包括：

1. **板块特征与网络特征**：衡量板块内部或产业链上的互动、联动、信息传导与协同效应
2. **网络互动捕捉**：通过图论方法捕捉"大哥涨、小弟跟"、"上游提价、下游承压"、"板块资金合力"的互动信号
3. **核心持仓聚焦**：模型训练聚焦4只核心持仓股，扩展股提供网络上下文和样本量

### 1.2 股票池架构

| 类型 | 数量 | 用途 |
|------|------|------|
| 核心持仓 | 4只 | 监控、预测、交易 |
| 扩展股票 | 49只 | 网络分析、样本扩充、产业链锚点 |
| **训练股票池** | **53只** | 模型训练 |

## 三、市场基准设计

### 3.1 双指数架构

放弃传统的上证指数或沪深300，采用"2 + X"模式：

| 指数 | 代码 | 角色 | 对应持仓 |
|------|------|------|---------|
| **中证1000** | 000852 | 全局风格与流动性基准 | 70%扩展股同频特征 |
| **创业板指** | 399006 | 成长股风险偏好基准 | 石药创新等创业板持仓 |

### 3.2 特征工程实现

```python
# 1. 强弱对比因子（Relative Strength）
df['RS_CSI1000'] = df['Return'] - df['CSI1000_Return']
# 作用：学习个股是"逆市吸筹"还是"跟风弱势"

# 2. 市场情绪互动（Market Sentiment Shift）
df['Sentiment_Ratio'] = df['CYB_Volume_5d'] / df['CSI1000_Volume_5d']
# 作用：捕捉资金往创业板集中的信号

# 3. 宏观利率对冲
df['CSI1000_US10Y_Spread'] = df['CSI1000_Return'] * df['US_10Y_Yield_Change']
# 作用：切分"全球降息 -> 中小盘成长弹性释放"规律
```

### 3.3 设计原理

**为什么选双指数而非单指数？**

| 指数 | 问题 |
|------|------|
| 上证指数 | 过于大盘股主导，与中小盘持仓特征偏离 |
| 沪深300 | 同样被银行/白酒等大盘股主导 |
| 创业板指单用 | 无法覆盖主板化工股（渤海化学） |

**双指数优势**：
- 中证1000：提供全市场中小盘Beta信号
- 创业板指：捕捉成长股风险偏好，直接触发高Risk节点级联效应

## 四、训练权重设计

### 4.1 混合权重策略

| 层面 | 权重配置 | 原因 |
|------|---------|------|
| **网络构建（图矩阵）** | 等权或按相关性加权 | 保持客观的资金联动关系 |
| **CatBoost训练（样本权重）** | 核心股3.0，扩展股1.0 | 强制模型聚焦核心持仓 |

### 4.2 代码实现

```python
# 网络层面：保持等权
# 原因：人为调高核心股权重会扭曲真实信息流向

# 训练层面：样本权重
A_STOCK_WEIGHTS = {
    '300440': 3.0,  # 运达科技
    '002655': 3.0,  # 共达电声
    '300765': 3.0,  # 石药创新
    '600800': 3.0,  # 渤海化学
    'default': 1.0  # 扩展股
}

# 训练时传入权重
df['sample_weight'] = df['is_core_holding'].apply(
    lambda x: 3.0 if x == 1 else 1.0
)

model.fit(
    X_train, y_train,
    sample_weight=df_train['sample_weight'],
    cat_features=categorical_features_indices
)
```

### 4.3 设计原理

**为什么不按市值/流动性加权？**

如果按市值或成交额加权，万华化学、中国石油、恒瑞医药等超级大盘股会"淹没"渤海化学（30亿）、运达科技（80亿）等核心股。网络特征将完全被大盘周期股主导。

**为什么不用纯等权？**

49只扩展股是"工具人"（提供样本量和网络上下文）。完全等权会让模型花92%（49/53）精力优化不打算买的股票，泛化掉核心股的独特性。

**样本权重3.0的效果**：
- 模型猜错核心股的惩罚是扩展股的3倍
- CatBoost在分裂树节点时，优先寻找契合核心股特征的路径
- 扩展股的网络交互特征被完美保留

## 九、潜在风险与避坑指南

> ⚠️ **重要**：以下风险点由专业评审指出，必须在落地时严格规避。

### 9.1 网络社区ID的时序泄漏隐患

**风险描述**：

图网络结构（如通过 Louvain 或 Infomap 算法切分的社区）在 A 股是**动态漂移**的。如果用过去3年的总数据跑出一个静态的社区 ID 喂给模型，在训练集的前期（如2023年），模型就等于提前知道了2026年的社区结构，属于隐性**时序数据泄漏**。

**修正方案**：

不要把"社区 ID"作为静态特征。如果一定要用，必须使用**滚动窗口（Rolling Window）**计算：

```python
# 正确做法：滚动窗口计算社区ID
def calculate_rolling_community(df, window=60):
    """使用前60天数据计算社区，避免前瞻性偏差"""
    for date in df.index:
        lookback_start = date - timedelta(days=window)
        lookback_data = df.loc[lookback_start:date]
        # 使用 lookback_data 计算相关性矩阵
        # 构建网络，划分社区
        # 将社区ID赋给 date + 1天（.shift(1)）
```

### 9.2 交叉特征的"过度共振"风险

**风险描述**：

CatBoost 自身具备极强的非线性特征组合能力。人工引入乘法交叉特征，如果处理不当，容易导致局部梯度爆炸或过拟合。

**修正方案**：

把网络特征和市场特征作为**独立特征**喂给 CatBoost，设置 `depth=6`，让树自己学习交叉逻辑：

```python
# 不推荐：硬编码乘法
# df['net_centrality_CSI1000'] = df['net_composite_centrality'] * df['CSI1000_Return_5d']

# 推荐：作为独立特征，让模型学习
features = ['net_composite_centrality', 'CSI1000_Return_5d', ...]
model = CatBoostRegressor(depth=6, ...)
```

### 9.3 主板与创业板的涨跌幅截断问题

**风险描述**：

模型混合了10%涨跌停（主板）与20%涨跌停（创业板）的股票。在预测未来N日收益率作为标签时，创业板股票的极端值会主导 MSE 损失函数。

**修正方案**：

在计算标签 $y$ 时使用**标准化收益率**：

```python
# 方案1：保留市场类型作为分类特征
df['market_type'] = df['stock_code'].apply(
    lambda x: 'gem' if x.startswith('300') else 'main'
)

# 方案2：标签标准化（除以滚动波动率）
df['label_normalized'] = df['future_return'] / df['volatility_20d']
```
