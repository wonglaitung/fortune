## 一、系统概述

### 1.1 设计目标

基于港股智能分析系统的成熟架构，构建A股版本，核心能力包括：

| 能力 | 说明 |
|------|------|
| **网络互动捕捉** | 通过图论方法捕捉"大哥涨、小弟跟"、"上游提价、下游承压"、"板块资金合力"的互动信号 |

| **核心持仓聚焦** | 模型训练聚焦4只核心持仓股，扩展股提供网络上下文和样本量 |

### 3.2 行为金融因子（业界最佳实践）

**研究来源**：
- 国信证券：《凸显性因子：行为金融学在A股的创新应用》
- 方正证券：《球队硬币因子：基于体育博彩理论的选股因子》

#### 凸显性因子 (STR)

基于 Kahneman-Tversky 前景理论：投资者对极端收益/损失给予过多关注。

```python
# 计算公式
STR_Nd = (Max_Return_Nd - Min_Return_Nd) / Avg_Abs_Return_Nd

# 特征列表
STR_5d          # 5日凸显性
STR_20d         # 20日凸显性
STR_Intensity   # 凸显强度 = STR_5d / STR_20d
STR_Signal      # 凸显信号（综合排名）
```

#### 球队硬币因子

基于体育博彩理论：当市场情绪过度偏向一方时，反向操作有利可图。

```python
# 计算公式
Up_Ratio = Up_Days / Total_Days  # 上涨天数占比
Team_Coin_Sentiment = (Up_Ratio - 0.5) * 2  # 偏离中性程度

# 特征列表
Team_Coin_Sentiment   # 市场情绪偏向 [-1, 1]
Team_Coin_Reversal    # 反转信号
Team_Coin_Intensity   # 情绪强度
```

**实现位置**：`ml_services/feature_engineering/behavioral_factors.py`

### 3.3 日内特征

学术验证：隔夜收益与日内收益对次日走势有显著预测力。

```python
Overnight_Return           # 隔夜收益（收盘 → 次日开盘）
Intraday_Return            # 日内收益（开盘 → 收盘）
Overnight_Intraday_Ratio   # 隔夜/日内收益比率
Open_Close_Ratio           # 开盘收盘关系
Consecutive_Up_Days        # 连续阳线天数
Consecutive_Down_Days      # 连续阴线天数
```

### 3.4 跨市场联动特征

**数据源**：AKShare（铜期货 CU0、原油期货 SC0、人民币汇率）

```python
Copper_Return_5d    # 铜期货5日收益（化工板块正相关）
Copper_Return_20d   # 铜期货20日收益
Oil_Return_5d       # 原油期货5日收益（化工成本端传导）
Oil_Return_20d      # 原油期货20日收益
CNY_USD_Change_5d   # 人民币汇率5日变化
CNY_USD_Z_Score     # 人民币汇率Z分数
```

**实现位置**：`data_services/a_stock_market_features.py`

### 3.5 特征单调性设计

交叉特征时保持逻辑单调性：

| 交叉类型 | 交叉方式 | 示例 |
|---------|---------|------|
| 正向 × 正向 | 乘法 | 中心性 × 收益率 |
| 负向 × 负向 | 风险放大 | 约束度 × VIX → `-|X| × |Y|` |
| 正向 × 负向 | 风险调整 | 中心性 × VIX → `X / (|Y| + ε)` |

### 3.6 数据泄漏防护

**高风险特征必须使用 `.shift(1)`**：

| 风险点 | 防护措施 |
|---------|---------|
| 所有 `.rolling()` 计算的特征 | 使用 `.shift(1)` 避免当日数据 |
| 涨跌停特征 | 使用前一日收盘价计算 |
| 北向资金特征 | 使用滞后数据 |
| 行为金融因子 | 使用滞后数据 |

```python
# 正确示例
df['STR_5d'] = (df['Max_Return_5d'] - df['Min_Return_5d']) / df['Avg_Abs_Return_5d']
df['STR_5d'] = df['STR_5d'].shift(1)  # 避免数据泄漏
```

## 四、模型训练

### 4.1 CatBoost 参数配置

| 参数 | 值 | 说明 |
|------|-----|------|
| n_estimators | 400 | 树的数量 |
| depth | 8 | 树深度 |
| learning_rate | 0.06 | 学习率 |
| l2_leaf_reg | 2 | L2正则化 |
| subsample | 0.75 | 子采样率 |
| colsample_bylevel | 0.8 | 列采样率 |
| random_seed | 42 | 固定随机种子 |

### 4.2 样本权重策略

**为什么需要样本权重？**

49只扩展股是"工具人"（提供样本量和网络上下文）。完全等权会让模型花92%（49/53）精力优化不打算买的股票。

**权重配置**：

```python
A_STOCK_WEIGHTS = {
    '300440': 3.0,  # 运达科技
    '002655': 3.0,  # 共达电声
    '300765': 3.0,  # 石药创新
    '600800': 3.0,  # 渤海化学
    'default': 1.0  # 扩展股
}

# 训练时传入权重
model.fit(X_train, y_train, sample_weight=df['sample_weight'])
```

**效果**：
- 模型猜错核心股的惩罚是扩展股的3倍
- CatBoost在分裂树节点时，优先寻找契合核心股特征的路径

### 4.3 标签定义

```python
# 未来N日累积收益率
future_return = returns.rolling(horizon).sum().shift(-horizon)

# 标准化标签（除以滚动波动率，解决主板/创业板涨跌幅差异）
label_normalized = future_return / volatility_20d

# 二元标签（收益 > 0）
label = (future_return > 0).astype(int)
```

|------|---------|-------------|

### 6.1 报告结构

| **市场环境分析** | 上证指数技术分析、北向资金趋势（5日/20日累积、连续流入） | ✅ v3.0 增强 |
| **板块分析** | 板块涨跌幅排名、龙头股TOP 3、板块类型（周期/防御） | ✅ v3.0 新增 |
| **异常检测提醒** | 三级严重度分类（高/中/低）、异常原因详细说明 | ✅ v3.0 新增 |

| **风险控制建议** | 市场风险评估、总仓位建议、止损策略 | ✅ v3.0 新增 |
| **AI 分析建议** | 通义千问大模型完整建议 | ✅ 已有 |

### 6.2 综合买卖建议生成器

**文件位置**：`a_stock_recommendation_generator.py`

**决策逻辑**：

| 建议类型 | 条件 | 建议仓位 |
|---------|------|---------|
| **强烈买入** | 短期买入 + 中期买入 + CatBoost概率 ≥ 0.55 | 4% |
| **买入** | 短期买入 + 中期买入 + 0.50 < CatBoost概率 < 0.55 | 3% |
| **持有/观望** | 其他情况 | 0% |
| **卖出** | 短期卖出 + 中期卖出 + CatBoost概率 ≤ 0.45 | - |

**价格指引**：
- 建议买入价：当前价或技术支撑位
- 止损位：-8%
- 目标价：+10%

**风险控制输出**：
- 市场风险评估：基于北向资金流向判断（高/中/低）
- 总仓位建议：强烈买入 + 买入仓位之和
- 止损策略：单只股票最大亏损不超过-8%

### 6.3 异常检测模块

**检测维度**：

| 维度 | 阈值 | 严重度 |
|------|------|--------|
| **RSI 超买/超卖** | RSI > 80 或 RSI < 20 | 高 |
| **RSI 超买/超卖** | RSI > 70 或 RSI < 30 | 中 |
| **大涨/大跌** | 接近涨跌停（±9.5%） | 高 |
| **大涨/大跌** | 单日涨跌幅 ≥ 5% | 中 |
| **成交量异常** | 量比 > 3 倍 | 中 |
| **成交量异常** | 量比 > 2 倍 | 低 |
| **价格偏离** | 偏离 MA20 > 15% | 中 |

**输出格式**：
- 高严重度：🔴 红色标记，需立即关注
- 中严重度：⚠️ 黄色标记，需关注
- 低严重度：📋 灰色标记，轻微异常

### 6.4 板块分析模块

**聚合逻辑**：
1. 按 `A_STOCK_SECTOR_MAPPING` 中的 `sector` 字段聚合
2. 计算板块平均涨跌幅
3. 找出板块内涨幅最大的前3只股票（龙头股）

## 七、风险与避坑

### 7.1 网络社区ID的时序泄漏

**风险**：用过去3年总数据跑出静态社区ID，在训练前期就知道未来社区结构。

**修正方案**：使用滚动窗口计算社区ID

```python
def calculate_rolling_community(df, window=60):
    for date in df.index:
        lookback_data = df.loc[date - timedelta(days=window):date]
        # 使用 lookback_data 计算相关性矩阵，构建网络
```

### 6.2 交叉特征的"过度共振"

**风险**：CatBoost自带非线性组合能力，人工乘法交叉可能过拟合。

**修正方案**：把网络特征和市场特征作为独立特征，设置 `depth=6`，让树自己学习交叉。

### 6.3 主板与创业板涨跌幅截断

**风险**：模型混合10%涨跌停（主板）与20%涨跌停（创业板），创业板极端值主导MSE。

**修正方案**：标签标准化（除以滚动波动率）

```python
df['label_normalized'] = df['future_return'] / df['volatility_20d']
```
