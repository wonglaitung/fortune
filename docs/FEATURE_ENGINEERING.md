## 一、核心原则

### 1.1 标签定义决定模型学什么

**核心结论**：截面问题用相对标签学 Alpha，时序问题用绝对标签学 Beta/方向。**混用等于让模型"答非所问"**。

| 问题类型 | 标签类型 | 模型学习目标 | 适用场景 |
|----------|----------|--------------|----------|
| **截面问题** | 相对标签 | 学 Alpha（跑赢其他股票） | 选股 |
| **时序问题** | 绝对标签 | 学 Beta/方向（涨还是跌） | 择时 |

### 1.2 模型目标与标签匹配

| 模型类型 | 目标 | 标签类型 | 标签定义 |
|----------|------|----------|----------|
| **个股模型** | 选股（挑出跑赢的股票） | **相对标签** | `Return > Median_Return` |
| **恒指模型** | 择时（预测大盘涨跌） | **绝对标签** | `Return > 0` |

**禁止混用**：
- ❌ 个股模型用绝对标签 → 模型变成"宏观择时器"，无法选股
- ❌ 恒指模型用相对标签 → 无意义，恒指没有比较对象

### 1.3 港股市场的特殊性

港股是典型的**离岸市场**，其定价权在美债利率（分母端）和内地基本面（分子端）之间拉锯。

| 维度 | 说明 | 对模型的影响 |
|------|------|--------------|
| **分母端（美债利率）** | 港股定价受美债利率高度影响 | 利率上升→估值压缩→普跌 |
| **分子端（内地基本面）** | 盈利依赖内地经济 | 经济周期→行业分化 |
| **流动性敏感** | 离岸市场，资金流动剧烈 | 外资流入流出→全市场同涨同跌 |

**结论**：对于港股这种受美债利率和流动性高度敏感的市场，个股选股必须用相对标签。

## 二、快速参考

### 2.1 核心警告

| 警告 | 说明 |
|------|------|
| **数据泄漏** | Walk-forward准确率 >65%（个股）或 >80%（恒指）通常是数据泄漏信号 |
| **IC 负值** | IC < 0 表示选股能力有限，需启用单调约束/时间衰减/滚动百分位 |
| **特征缓存版本** | 新增特征后必须清除缓存（`rm -rf data/feature_cache/*.pkl`） |
| **分类特征 NaN** | CatBoost 预测时必须处理分类特征 NaN，训练和预测的预处理必须一致 |

### 2.2 新增特征检查清单

| 步骤 | 检查项 | 标准 |
|------|--------|------|
| 1 | **泄漏检查** | 所有特征使用 `shift(1)` |
| 2 | **单调约束** | 有明确方向逻辑则添加到 `MONOTONE_CONSTRAINT_MAP` |
| 3 | **滚动百分位** | 绝对量级特征添加到 `ROLLING_PERCENTILE_FEATURES` |
| 4 | **训练/预测一致性** | 特征变换在两处都执行 |
| 5 | **模型保存/加载** | 新增参数保存到 `save_model()` |

### 2.3 常用命令

```bash
# Walk-forward 验证
python3 ml_services/walk_forward_validation.py --model-type catboost --horizon 20

# 清除特征缓存
rm -rf data/feature_cache/*.pkl

# 语法检查
python3 -m py_compile ml_services/ml_trading_model.py
```

### 2.4 关键指标目标

| 指标 | 个股目标 | 恒指目标 |
|------|---------|---------|
| 准确率 | 50-65% | 60-82% |
| IC | >0.02 | >0.05 |
| 预测分散度 | >0.1 | >0.1 |

## 三、特征体系

### 3.1 特征数量统计

| 特征类别 | 数量 | 说明 |
|---------|------|------|
| **滚动统计特征** | 126 | 偏度、峰度、多周期波动率 |
| **价格形态特征** | 84 | 日内振幅、影线比例、缺口 |
| **量价关系特征** | 98 | 背离、OBV、成交量波动率 |
| **长期趋势特征** | 84 | MA120/250、长期收益率、长期RSI |
| **GARCH 波动率特征** | 4 | 条件波动率、波动率比率、波动率变化、持续性参数 |
| **HSI 市场状态特征** | 6 | HMM 市场状态检测（状态标签、概率、持续时间、转换概率） |
| **日历效应特征** | 22 | 星期效应、月份效应、期权到期日、月初/月末等 |
| **残差特征** | 35 | 特征相对市场残差（剔除宏观因子影响） |
| **交叉特征** | 360 | 8个类别 × 45个数值（已清理冗余） |
| **总计** | **770** | **含35个残差特征** |

### 3.2 特征分类原则

| 类别 | 说明 | 示例 |
|------|------|------|
| **时间序列特征** | 基于历史价格、成交量数据计算 | MA、RSI、MACD |
| **市场环境特征** | 基于市场状态、波动率等宏观指标 | HSI_Regime、VIX |
| **基本面特征** | 基于公司财务数据 | PE、PB、ROE |
| **残差特征** | 剔除宏观因子后的个股特异性特征 | Momentum_Residual |

### 3.3 核心特征类别详解

#### 3.3.1 残差特征（35个）

对关键微观特征进行残差化处理，剔除宏观因子影响。

**实现原理**：
```python
# 微观特征残差 = 微观特征 - β * 宏观特征
model.fit(X_macro, X_micro)
residual = X_micro - model.predict(X_macro)
```

**效果**：显著降低特征之间的共线性，迫使模型学习个股特异性。

**实现文件**：`data_services/feature_residualizer.py`

#### 3.3.2 GARCH 波动率特征（4个）

使用 GARCH(1,1) 模型捕捉波动率聚类特性。

| 特征名称 | 说明 |
|---------|------|
| `GARCH_Conditional_Vol` | 条件波动率 |
| `GARCH_Vol_Ratio` | 波动率比率（当前/历史均值） |
| `GARCH_Vol_Change_5d` | 5日波动率变化 |
| `GARCH_Persistence` | 波动率持续性参数（α + β） |

**实现文件**：`data_services/volatility_model.py`

#### 3.3.3 HSI 市场状态特征（6个）

使用 HMM（隐马尔可夫模型）识别恒生指数市场状态。

| 特征名称 | 说明 | 取值范围 |
|---------|------|---------|
| `HSI_Market_Regime` | 市场状态标签 | 0=震荡, 1=牛市, 2=熊市 |
| `HSI_Regime_Prob_0/1/2` | 各状态概率 | 0-1 |
| `HSI_Regime_Duration` | 当前状态持续时间 | 1-100+ 天 |

**特征重要性**：HSI_Regime_Duration 排名第2（3.95分）

**实现文件**：`data_services/regime_detector.py`

#### 3.3.4 日历效应特征（22个）

捕捉周期性市场规律：星期效应、月份效应、节假日效应、期权到期效应。

**实现文件**：`data_services/calendar_features.py`

## 四、问题诊断与解决方案

### 4.1 问题：全局特征导致"特征坍缩"

**现象**：个股模型预测结果"全涨全跌"，无法选股。

**原因**：美债利率等全局特征导致模型"偷懒"，只看宏观信号，忽略个股特异性。

#### 解决方案 1：相对标签（推荐）

```python
# 以当天所有股票收益率的中位数为基准
median_return = returns.median()
y = (returns > median_return).astype(int)
```

**效果**：美债利率权重下降，模型被迫学习个股特异性特征。

#### 解决方案 2：特征残差化

```python
# 剔除宏观因子贡献
residual = X_micro - model.predict(X_macro)
```

**效果**：显著降低特征之间的共线性。

#### 解决方案 3：标签残差化（Beta中性化）

```python
# 预测"是否跑赢宏观宿命"
expected_return = beta * macro_change
y = (actual_return > expected_return).astype(int)
```

### 4.2 方案对比

| 方案 | 实施位置 | 适用场景 | 复杂度 |
|------|----------|----------|--------|
| **相对标签** | 标签端 | 选股为主 | ⭐ 简单 |
| **特征残差化** | 特征端 | 降低共线性 | ⭐⭐ 中等 |
| **标签残差化** | 标签端 | 选股+择时分离 | ⭐⭐⭐ 复杂 |

### 4.3 问题：IC 负值

**现象**：IC < 0，预测概率高的股票实际收益反而低。

**原因**：Regime Shift 导致特征方向翻转。

| 问题 | 说明 |
|------|------|
| **特征方向不稳定** | 2020年"波动率↑ → 下跌"，2024年可能变成"波动率↑ → 上涨" |
| **旧数据干扰** | 2020-2022 年的数据权重过高 |
| **绝对量级失效** | 波动率=0.03 在不同时期含义不同 |

**解决方案**：

| 方案 | 参数 | 说明 |
|------|------|------|
| **单调约束** | `use_monotone_constraints=True` | 强制特征方向不变 |
| **时间衰减** | `time_decay_lambda=0.5` | 降低旧数据权重 |
| **滚动百分位** | `use_rolling_percentile=True` | 绝对值转历史百分位 |

**单调约束方向参考**：

| 特征类别 | 约束方向 | 理论依据 |
|----------|----------|----------|
| 波动率（Volatility, ATR, GARCH） | `-1` 递减 | 低波动异象 |
| 股息（Dividend_Yield） | `+1` 递增 | 股息溢价 |
| 相对强度（RS_Ratio） | `+1` 递增 | 动量持续 |

### 4.4 对模型指标的影响

| 指标 | 变化 | 原因 |
|------|------|------|
| **总准确率** | 可能下降 | "送分题"被删除，留下的是真正的 Alpha |
| **信噪比** | 显著提高 | 准确率由个股特异性贡献 |
| **预测一致性** | 变强 | 不再因宏观抖动导致全场预测翻转 |
| **选股能力** | 提升 | 模型被迫学习个股差异 |

## 五、最佳实践与检查清单

### 5.1 数据泄漏防护

**原则**：所有特征必须使用滞后数据

```python
# ❌ 错误：使用当日数据
df['Volume_Ratio'] = df['Volume'] / df['Volume'].rolling(5).mean()

# ✅ 正确：使用滞后数据
df['Volume_Ratio'] = df['Volume'].shift(1) / df['Volume'].shift(1).rolling(5).mean()
```

**高风险特征检查清单**：
- [ ] 所有 `.rolling()` 计算后是否有 `.shift(1)`
- [ ] 标签是否使用 `shift(-horizon)` 获取未来收益
- [ ] 价格距离/百分比特征是否使用滞后价格
- [ ] 宏观因子是否对齐到正确日期

### 5.2 新特征上线验证清单

| 步骤 | 验证项 | 标准（恒指） | 标准（个股） |
|------|--------|-------------|-------------|
| 1 | **泄漏检查** | 所有特征使用 `shift(1)` | 所有特征使用 `shift(1)` |
| 2 | **Walk-forward 验证** | 目标 ≥ 82%；> 85% 需审计 | 目标 ≥ 50%；> 65% 需审计 |
| 3 | **SHAP 排名** | 进入 top 30 | 进入 top 30 |
| 4 | **Pearson 相关性** | 与现有特征 < 0.8 | 与现有特征 < 0.8 |
| 5 | **随机种子稳定性** | 3 个种子波动 < 2% | 3 个种子波动 < 2% |
| 6 | **单调约束判断** | 有明确方向逻辑则添加 | 有明确方向逻辑则添加 |
| 7 | **滚动百分位判断** | 绝对量级特征需转换 | 绝对量级特征需转换 |
| 8 | **训练/预测一致性** | 特征变换在两处都执行 | 特征变换在两处都执行 |

### 5.3 特征相关性筛选阈值

**为什么是 0.8**：

| 阈值 | 问题 |
|------|------|
| 0.9 | 太松 — 81% 方差重叠，基本是同一信号 |
| 0.5 | 太严 — 0.5-0.8 之间可能包含互补信息 |
| **0.8** | **工程折中** — 保留足够独立的信号 |

### 5.4 特征冗余清理记录

| 清理类别 | 删除数量 | 原因 |
|---------|---------|------|
| RS_Signal 重复 | 251 | 与 Trend 公式完全相同 |
| PE/PB 常量交互 | 18 | PE/PB 为常量，交互无信息 |
| 数学等价特征 | 15 | Returns/Volatility/Momentum 等重复 |
| **合计** | **~307** | 特征数量 1045→730 |
