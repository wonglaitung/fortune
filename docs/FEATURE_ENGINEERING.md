## 特征工程最佳实践

### 1. 使用全量特征

**推荐命令**：
```bash
python3 ml_services/ml_trading_model.py --mode train --horizon 20 --model-type catboost
```

CatBoost 内置 L2 正则化和自动特征重要性计算，能够自动降权不重要特征，无需预先筛选。

### 2. 特征相关性筛选阈值（Pearson < 0.8）

**阈值选择**：新增特征与现有特征相关性应 **< 0.8**，超过则视为冗余。

**原因分析**：

| 问题 | 说明 |
|------|------|
| **信息量不增加** | 相关性 > 0.8 的特征本质上是同一信号的不同表达，CatBoost 无法获得额外信息，反而增加噪音 |
| **特征重要性分散** | 同一信号由两个高相关特征瓜分重要性，导致 SHAP 排名失真，误判哪个特征真正有用 |
| **过拟合风险** | 模型可能在不同 fold 中交替选择两个高相关特征，降低稳定性 |
| **SHAP 特征选择失效** | 新特征需进 top 30、最终特征集 < 50，冗余特征挤入会排挤真正有价值但独立的特征 |

**为什么是 0.8**：

| 阈值 | 问题 |
|------|------|
| 0.9 | 太松 — 81% 方差重叠，基本是同一信号 |
| 0.5 | 太严 — 0.5-0.8 之间可能包含互补信息（如 RSI 和 MACD 都衡量动量，但捕捉不同方面） |
| **0.8** | **工程折中** — 保留足够独立的信号，同时不丢弃有部分重叠但有增量价值的特征 |

**实践建议**：
- 新特征上线前检查与现有特征的 Pearson 相关系数
- 相关性 > 0.8 时，评估是否带来增量信息；若无，则丢弃
- CatBoost 对多重共线性容忍度高于线性模型，但高相关特征仍会影响特征重要性解释

### 3. 数据泄漏防护

**原则**：所有特征必须使用滞后数据

```python
# ❌ 错误：使用当日数据
df['Volume_Ratio'] = df['Volume'] / df['Volume'].rolling(5).mean()

# ✅ 正确：使用滞后数据
df['Volume_Ratio'] = df['Volume'].shift(1) / df['Volume'].shift(1).rolling(5).mean()
```

**新增特征的 shift 处理**：
```python
# GARCH 波动率特征
for col in ['GARCH_Conditional_Vol', 'GARCH_Vol_Ratio', 'GARCH_Vol_Change_5d']:
    df[col] = df[col].shift(1)

# HSI 市场状态特征
for col in ['Market_Regime', 'Regime_Prob_0', 'Regime_Prob_1', 'Regime_Prob_2',
            'Regime_Duration', 'Regime_Transition_Prob']:
    df[col] = df[col].shift(1)
```

**高风险特征检查清单**：
- [ ] 所有 `.rolling()` 计算后是否有 `.shift(1)`
- [ ] 标签是否使用 `shift(-horizon)` 获取未来收益
- [ ] 价格距离/百分比特征是否使用滞后价格
- [ ] 宏观因子是否对齐到正确日期

## 全局特征与标签设计

### 问题背景：全局特征的"作弊金手指"效应

在二分类任务中预测股票涨跌时，美债利率等全局特征会导致"概率倾斜"问题。

#### 1. "大潮效应"：模型学会了"偷懒"

当预测多只股票涨跌时，如果美债利率大幅变动，市场往往会出现"普涨"或"普跌"。

**逻辑**：
- 假设明天因利率大涨，80%的股票都要跌
- 模型发现："只要看美债利率这一个特征，预测所有股票都'跌'，就能拿到80%准确率"
- 后果：模型放弃挖掘个股动量、成交量特征，因为"看宏观风向标"的收益成本比最高

这在机器学习中被称为**特征坍缩（Feature Collapse）**。

#### 2. 标签高度相关：违反IID假设

| 问题 | 说明 |
|------|------|
| **标签同步化** | 美债利率把所有样本的标签"同步化"了 |
| **损失函数误导** | Cross-Entropy会惩罚预测错误的样本，若5000个样本标签全因利率变成0，模型会给利率特征极高权重 |
| **忽略个股信号** | 模型忽略那些可能在普跌中逆势上涨的个股信号 |

#### 3. 预测"升跌"比预测收益率更易受干扰

| 维度 | 说明 |
|------|------|
| **缺乏区分度** | 收益率有"涨多涨少"的区别，升跌只有0和1 |
| **信噪比问题** | 美债利率是高信噪比宏观信号，个股Alpha信号通常是低信噪比 |
| **模型偏好** | 模型天然喜欢高信噪比特征，即便该特征对选股毫无帮助 |

### 实战问题

#### 问题1：全仓买入或全仓空仓

模型给出的预测结果往往是"全场都在升"或"全场都在跌"，导致无法进行选股（Stock Picking），只能进行择时（Market Timing）。

#### 问题2：宏观变盘时的"踩踏"

当美债利率进入震荡期，或利率对股市的解释力下降时，模型会因为失去核心"拐杖"而变得像随机乱猜。

### 解决方案

#### 方案1：相对标签（Cross-sectional Labeling）

**核心思想**：预测"跑赢/跑输"，而非"绝对涨跌"

**实现方法**：
```python
# 不要以 0 为基准
y = (returns > 0).astype(int)

# 以当天所有股票收益率的中位数为基准
median_return = returns.median()
y = (returns > median_return).astype(int)
```

**效果**：
- 美债利率权重迅速下降（甚至变为0）
- 因为利率对所有股票作用一样，无法解释为什么A股票比中位数表现更好
- 模型被迫学习个股特异性特征

#### 方案2：特征端残差化

**核心思想**：剥离"宏观重力"

**实现方法**：
```python
from sklearn.linear_model import LinearRegression

# 对每个微观特征，剔除宏观因子贡献
def residualize_feature(X_micro, X_macro):
    """
    X_micro: 微观特征（如个股动量）
    X_macro: 宏观因子（如美债利率、VIX）
    """
    model = LinearRegression()
    model.fit(X_macro, X_micro)
    residual = X_micro - model.predict(X_macro)
    return residual

# 示例：剔除利率对动量的影响
momentum_residual = residualize_feature(
    df['Momentum_20d'].values.reshape(-1, 1),
    df[['US_10Y_Yield', 'VIX']].values
)
```

**效果**：
- 模型看到的动量不再是"因为利率下降而带来的普涨动量"
- 而是"剔除利率因素后，这只票特有的动量"
- 显著降低特征之间的共线性

#### 方案3：标签端残差化（Beta中性化）

**核心思想**：预测"是否跑赢宏观宿命"

**实现方法**：
```python
# 1. 计算个股的宏观敏感度（β）
from sklearn.linear_model import LinearRegression

def calculate_beta(stock_returns, macro_changes):
    """计算个股对宏观因子的敏感度"""
    model = LinearRegression()
    model.fit(macro_changes.reshape(-1, 1), stock_returns)
    return model.coef_[0]

# 2. 构建预期收益
beta = calculate_beta(stock_returns, us_10y_changes)
expected_return = beta * us_10y_change

# 3. 重新定义标签
y = (actual_return > expected_return).astype(int)
```

**逻辑变更**：
- 预测的不是"明天这只票涨不涨"
- 而是"明天这只票是否跑赢了由宏观环境决定的'宿命'"

### 方案对比

| 方案 | 实施位置 | 核心思想 | 适用场景 | 复杂度 |
|------|----------|----------|----------|--------|
| **相对标签** | 标签端 | 以中位数为基准 | 选股为主 | ⭐ 简单 |
| **特征残差化** | 特征端 | 剔除宏观对微观特征的影响 | 降低共线性 | ⭐⭐ 中等 |
| **标签残差化** | 标签端 | 预测"跑赢宏观宿命" | 选股+择时分离 | ⭐⭐⭐ 复杂 |

### 对模型指标的影响

| 指标 | 变化 | 原因 |
|------|------|------|
| **总准确率** | 可能下降 | 之前靠美债利率能轻松猜对80%的集体涨跌，现在这部分"送分题"被删除 |
| **信噪比** | 显著提高 | 留下的准确率由个股特异性（Alpha）贡献，实盘中更稳健 |
| **预测一致性** | 变强 | 模型不再因美债利率抖动导致全场预测结果集体翻转 |
| **选股能力** | 提升 | 模型被迫学习个股差异，而非依赖宏观信号 |

### 实施建议

#### 短期方案：相对标签

1. 修改 `ml_trading_model.py` 中的标签生成逻辑
2. 以当天所有股票收益率的中位数为基准
3. 验证指标增加 IC（Information Coefficient）和 Rank IC

#### 中期方案：特征残差化

1. 对动量、成交量等微观特征，剔除宏观因子贡献
2. 保留宏观特征作为独立输入（用于择时判断）
3. 监控特征相关性变化

#### 验证指标调整

除了准确率，建议增加：

| 指标 | 计算方法 | 说明 |
|------|----------|------|
| **IC** | 预测值与实际收益率的Pearson相关系数 | 衡量预测能力 |
| **Rank IC** | 预测排名与实际收益排名的Spearman相关系数 | 衡量排序能力 |
| **预测分散度** | 预测结果的标准差 | 避免"全涨全跌" |

#### 方案评价：逻辑闭环了吗？

| 方案 | 评价 | 关键洞察 |
|------|------|----------|
| **短期方案（相对标签）** | 性价比最高 | 直接在数学上"杀掉"宏观重力，强迫模型在每一天的横截面上找差异 |
| **中期方案（特征残差化）** | 工程上最稳健 | 构建条件模型（Conditional Model）：在当前利率环境下，个股的网络拓扑意味着什么 |
| **验证指标调整** | 质的飞跃 | 准确率在量化里经常骗人，Rank IC 能告诉你模型对股票"好坏排序"的真实能力 |

#### 为什么在港股背景下，这套方案尤为重要？

港股是典型的**离岸市场**，其定价权在美债利率（分母端）和内地基本面（分子端）之间拉锯。

**不实施相对标签的后果**：
- 模型变成"美债利率预测器"
- 在港股这种波动剧烈的市场，频繁给全场2000多只股票发"全卖"或"全买"信号
- 实盘中完全没法调仓

**港股的特殊性**：

| 维度 | 说明 | 对模型的影响 |
|------|------|--------------|
| **分母端（美债利率）** | 港股定价受美债利率高度影响 | 利率上升→估值压缩→普跌 |
| **分子端（内地基本面）** | 盈利依赖内地经济 | 经济周期→行业分化 |
| **流动性敏感** | 离岸市场，资金流动剧烈 | 外资流入流出→全市场同涨同跌 |

**结论**：对于港股这种受美债利率和流动性高度敏感的市场，相对标签是找回 Alpha 的第一步。
