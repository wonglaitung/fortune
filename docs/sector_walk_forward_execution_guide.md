# 板块Walk-forward验证执行指南

**生成时间**: 2026-03-16 03:15
**脚本版本**: v1.0

## 📋 概述

本指南详细说明了如何执行板块Walk-forward验证，包括运行脚本、输入参数、输出文件等内容。

## 🚀 执行步骤

### 1. 验证所有板块（推荐）

**命令**：
```bash
python3 ml_services/walk_forward_by_sector.py --all-sectors --start-date 2024-01-01 --end-date 2025-12-31 --train-window 12 --test-window 1 --step-window 1 --horizon 20 --confidence-threshold 0.55 --use-feature-selection
```

**参数说明**：
- `--all-sectors`: 验证所有16个板块
- `--start-date 2024-01-01`: 验证开始日期
- `--end-date 2025-12-31`: 验证结束日期
- `--train-window 12`: 训练窗口12个月（业界标准）
- `--test-window 1`: 测试窗口1个月（业界标准）
- `--step-window 1`: 滚动步长1个月（业界标准）
- `--horizon 20`: 预测周期20天
- `--confidence-threshold 0.55`: 置信度阈值55%
- `--use-feature-selection`: 使用特征选择（500个精选特征）

**执行时间**：约4小时（16个板块 × 每个板块约15分钟）

### 2. 验证单个板块

**命令**：
```bash
python3 ml_services/walk_forward_by_sector.py --sector bank
```

**支持的板块**：
- `bank`: 银行股
- `tech`: 科技股
- `semiconductor`: 半导体股
- `ai`: 人工智能股
- `new_energy`: 新能源股
- `environmental`: 环保股
- `energy`: 能源股
- `shipping`: 航运股
- `exchange`: 交易所
- `utility`: 公用事业股
- `insurance`: 保险股
- `biotech`: 生物医药股
- `index`: 指数基金
- `real_estate`: 房地产股
- `consumer`: 消费股
- `auto`: 汽车股

## 🔍 验证流程

### Walk-forward验证步骤

1. **数据准备**：
   - 获取股票历史数据（价格、成交量、技术指标）
   - 获取基本面数据（PE、PB、ROE等）
   - 获取美股市场数据（标普500、纳斯达克、VIX、国债收益率）
   - 生成主题情感交互特征（50个）
   - 生成预期差距特征（5个）
   - 生成交叉特征（3861个）
   - 应用特征选择（500个精选特征）

2. **Fold 1**：
   - 训练期：2024-01-01 至 2024-12-31（12个月）
   - 测试期：2025-01-01 至 2025-01-31（1个月）
   - 训练模型
   - 生成预测
   - 计算评估指标

3. **Fold 2**：
   - 训练期：2024-02-01 至 2025-01-31（12个月）
   - 测试期：2025-02-01 至 2025-02-28（1个月）
   - 训练模型
   - 生成预测
   - 计算评估指标

4. **重复至Fold 12**：
   - 每次滚动1个月
   - 总共12个fold

5. **计算整体指标**：
   - 平均收益率、胜率、准确率
   - 夏普比率、索提诺比率、信息比率
   - 最大回撤、稳定性分析

6. **生成报告**：
   - 保存JSON、CSV、Markdown格式报告

## 📈 评估指标

### 基础指标
- **平均收益率**：策略的平均收益
- **胜率**：盈利交易占比
- **准确率**：预测方向正确的比例
- **样本数**：每个fold的样本数量

### 风险指标
- **收益率标准差**：收益率波动性
- **年化收益率**：年化后的收益率
- **年化标准差**：年化后的波动性
- **最大回撤**：最大损失幅度

### 风险调整收益
- **夏普比率**：（收益率 - 无风险利率）/ 标准差
  - >1：优秀
  - >0.5：良好
  - >0：可接受
  - <0：不佳
- **索提诺比率**：（收益率 - 无风险利率）/ 下行标准差
  - >2：优秀
  - >1：良好
- **信息比率**：超额收益 / 跟踪误差

### 稳定性分析
- **收益率标准差**：衡量收益率波动
- **收益率范围**：最大收益率 - 最小收益率
- **胜率标准差**：衡量胜率波动
- **夏普比率标准差**：衡量夏普比率波动
- **稳定性评级**：
  - 高（优秀）：收益率标准差 < 2%
  - 中（良好）：收益率标准差 < 5%
  - 低（需改进）：收益率标准差 >= 5%

## ⚠️ 注意事项

### 1. 执行时间
- 验证所有16个板块：约4小时
- 验证单个板块：约15分钟
- 建议：使用后台运行或nohup

### 2. 内存要求
- 特征工程阶段：约4-8GB RAM
- 模型训练阶段：约2-4GB RAM
- 建议：至少8GB可用内存

### 3. 数据依赖
- 需要股票历史数据
- 需要基本面数据缓存
- 需要新闻数据（主题情感分析）
- 首次运行需要较长时间（数据下载和缓存）

### 4. 验证结果解读
- 夏普比率 > 0：推荐
- 夏普比率 > 0.2：强烈推荐
- 稳定性评级"高"：收益率波动小
- 稳定性评级"高"不代表收益好（需要看夏普比率）

## 🔧 故障排除

### 问题1：特征选择文件不存在
**错误信息**：
```
FileNotFoundError: [Errno 2] No such file or directory: 'output/selected_features_20d_statistical.csv'
```

**解决方案**：
```bash
# 先运行特征选择
python3 ml_services/feature_selection.py --method statistical --top-k 500 --horizon 20 --output-dir output

# 然后再运行Walk-forward验证
python3 ml_services/walk_forward_by_sector.py --all-sectors
```

### 问题2：内存不足
**错误信息**：
```
MemoryError: Unable to allocate array
```

**解决方案**：
- 关闭其他程序释放内存
- 减少验证板块数量
- 使用 `--use-feature-selection` 减少特征数量

### 问题3：数据下载失败
**错误信息**：
```
ConnectionError: Failed to download stock data
```

**解决方案**：
- 检查网络连接
- 使用VPN或代理
- 使用缓存数据（已有数据会自动使用缓存）
