## 模型相关类

### 时间序列数据泄漏警告

> **高准确率（>65%）通常是数据泄漏的信号**

**检查要点**：
- 检查数据合并时是否使用了 `ignore_index=True`
- 确保日期索引被保留，数据按时间顺序排列
- 验证时间序列交叉验证是否严格按时间顺序分割

**准确率参考范围**：
| 等级 | 准确率范围 |
|------|-----------|
| 随机/平衡基线 | ≈50% |
| 常见弱信号 | ≈51–55% |
| 有意义的改进 | ≈55–60% |
| 非常好/罕见 | ≈60–65% |
| 异常高（需怀疑） | >65% |

### CatBoost 1天模型过拟合警告

> **CatBoost 1天模型存在严重过拟合风险，不推荐使用**

**问题**：
- 准确率 63.09%（±4.33%），标准偏差过高
- 准确率甚至高于 CatBoost 5天和20天，违反一般规律

**根本原因**：样本量差异导致短期波动噪声被过度学习

**模型可信度评估**：
- CatBoost 20天：高可信度，推荐使用 ⭐⭐⭐⭐⭐
- CatBoost 5天：中等可信度，谨慎使用 ⭐⭐⭐
- CatBoost 1天：低可信度，不推荐 ⭐

### 融合模型信号稀释警告

> **融合模型表现不如 CatBoost 单模型，建议优先使用 CatBoost 单模型**

**回测对比**（置信度 0.55，28只股票）：
| 模型 | 年化收益率 | 夏普比率 | 买入信号胜率 |
|------|-----------|---------|-------------|
| CatBoost 单模型 | 79.54% | 1.14 | 30.96% |
| 融合模型（加权平均） | 3.16% | 0.32 | 22.75% |

**根本原因**：CatBoost 识别高质量强信号，GBDT/LightGBM 识别低质量弱信号，融合导致强信号被稀释

### 深度学习模型失效警告

> **深度学习模型（LSTM、Transformer）表现远不如 CatBoost，不适合用于金融时序预测**

**实验结果**（1天预测）：
| 模型 | 准确率 | F1分数 | 年化收益率 |
|------|--------|--------|-----------|
| CatBoost | 63.09% | 0.6022 | 79.54% |
| LSTM | 51.79% | **0.0000** | 0.00% |
| Transformer | 51.15% | 0.1303 | 0.00% |

**失败根本原因**：
1. 数据量不足（单个股票约700个样本，需要>10,000个）
2. 信噪比极低（<10%）
3. 特征工程不足（使用原始价格序列 vs CatBoost的500个精选特征）
4. 序列依赖假设错误（股价主要是随机游走）
5. 过拟合风险高（参数/样本比>140:1）

## 收益率计算类

### 夏普比率计算错误

> **夏普比率计算缺少无风险利率，导致结果严重低估**

**标准公式**：Sharpe = (年化收益率 - 无风险利率) / 年化标准差

**常见错误**：
1. 缺少无风险利率（通常取2%）
2. 使用20天收益率而非年化收益率
3. 使用所有样本的标准差（包含非买入信号的0值）

**正确做法**：
```python
risk_free_rate = 0.02  # 2%无风险利率
annualized_return = avg_return * (252 / horizon)  # 年化
annualized_std = return_std * np.sqrt(252 / horizon)
sharpe_ratio = (annualized_return - risk_free_rate) / annualized_std
```

### 收益率指标定义警告

> **收益率指标定义必须明确，避免报告误导**

**20天持有期收益率**：每次交易持有20天的收益率
**年化收益率**：20天收益率 × 12.6

**报告要求**：必须同时显示20天收益率和年化收益率，明确说明持有期

## 关键指标理解类

### 胜率 vs 准确率 vs 正确决策比例

| 指标 | 定义 | 使用场景 |
|------|------|----------|
| **胜率** | 盈利交易数 / 买入信号总数 | 评估买入信号质量 |
| **准确率** | 预测正确数 / 总交易机会数 | 评估方向预测能力 |
| **正确决策比例** | 正确决策数 / 总决策数 | 综合评估决策质量 |

### 高准确率不等于高收益

**案例**：准确率86.21%，买入信号胜率23.04%，平均收益率-1.94%

**原因**：模型正确识别下跌机会（高准确率），但买入信号质量差（低买入信号胜率）

**投资启示**：
1. 不能只看准确率，必须同时关注胜率和收益率
2. 在熊市中，模型的价值在于"避雷"而非"捕猎"
3. 需要根据市场环境动态调整策略

## 模型评估方法错误

### 简单评估方法存在数据泄漏

> **使用训练数据评估模型会导致结果虚高，必须使用严格的时序验证**

**错误做法**：
```python
y_pred = model.catboost_model.predict(X)  # 使用所有数据（包括训练数据）
accuracy = accuracy_score(y, y_pred)  # 结果虚高（83-90%）
```

**正确方法**：Walk-forward 验证（业界标准）
- 每个fold重新训练模型
- 严格的时序分割（避免数据泄漏）
- 评估真实预测能力

**参数配置**：训练窗口12个月，测试窗口1个月，滚动步长1个月

### 避免数据泄漏的最佳实践

**DO（应该做）**：
1. ✅ 使用 Walk-forward 验证
2. ✅ 时间序列分割（TimeSeriesSplit）
3. ✅ 每个fold重新训练模型
4. ✅ 独立的测试集（不参与训练）
5. ✅ 严格的日期顺序分割

**DON'T（不应该做）**：
1. ❌ 使用训练数据评估模型
2. ❌ 随机分割时间序列数据
3. ❌ 使用预测数据作为特征
4. ❌ 在分割前进行全局归一化
5. ❌ 使用未来信息预测过去

## 多周期策略回撤计算错误与修正

> **多周期（horizon>1）持有策略必须使用非重叠样本计算回撤，避免重复计算和复利放大**

### 问题描述

**Walk-forward验证显示银行股回撤高达-65.45%，严重偏离银行股低风险特性**

**根本原因**：原代码对多周期持有策略使用重叠样本计算回撤，导致收益被重复计算和复利放大

### 正确做法

```python
# 对于多周期预测(horizon>1)，使用非重叠样本
if self.horizon > 1:
    non_overlapping = df.iloc[::self.horizon].copy()
    cumulative_returns = (1 + non_overlapping['strategy_return']).cumprod()
    peak = cumulative_returns.expanding(min_periods=1).max()
    drawdown = (cumulative_returns - peak) / peak
    max_drawdown = drawdown.min()
```

### 验证结果

| 指标 | 修正前 | 修正后 | 变化 |
|------|--------|--------|------|
| 平均最大回撤 | -65.45% | **-13.12%** | **+52.33%** ✅ |
| 夏普比率 | 0.0525 | **0.1546** | +0.1021 ✅ |
| 年化收益率 | 37.71% | **45.92%** | +8.21% ✅ |

**关键洞察**：
- 回撤回归合理水平（-13.12%符合银行股低风险特性）
- 极端回撤消失（原本-90%以上的极端回撤被修正）
- 夏普比率大幅提升（风险调整收益显著改善）

## 数据泄漏检查与修复

### 存量特征系统性检查

> **使用当日OHLC数据计算的特征普遍存在数据泄漏风险**

**发现的高风险特征**（8个）：
1. BB_Position/BB_Breakout：使用当日布林带Upper/Lower
2. Price_Percentile：使用当日Close计算百分位
3. Intraday_Amplitude：使用当日High/Low计算振幅
4. Price_Ratio_MA5/20/50：使用当日Close计算比率
5. Price_Pct_20d：使用当日Close计算百分位
6. Days_Since_High/Low_20d：使用当日Close判断高低点
7. Intraday_Range：使用当日High/Low
8. Support_120d/Resistance_120d：使用rolling含当天数据

**修复方法**：
```python
# 修复前（数据泄漏）
df['BB_Position'] = (df['Close'] - df['BB_lower']) / (df['BB_upper'] - df['BB_lower'])

# 修复后（无泄漏）
df['BB_Position'] = (df['Close'] - df['BB_lower'].shift(1)) / (df['BB_upper'].shift(1) - df['BB_lower'].shift(1) + 1e-10)
```

**修复前后性能对比**：
| 指标 | 修复前 | 修复后 | 变化 |
|------|--------|--------|------|
| 准确率 | 61.48% | 60.79% | -0.69% |
| 标准差 | ±1.91% | ±2.44% | +0.53% |
| F1分数 | 0.6403 | 0.6211 | -0.0192 |

**关键发现**：
- 性能下降是正常的，反映真实预测能力
- 泛化能力提升，长期稳定性更强

## 置信度阈值优化经验

### 置信度阈值从0.55提升至0.6的效果分析

## 特征工程类

### 累积重要性特征选择的局限性

> **特征分数分布极其分散，累积重要性方法难以达到高覆盖率**

**实验结果**（2026-03-23）：
- 总特征数：4994个
- 有效特征数：4833-4974个
- F-test分数分布：极度分散，无明显拐点

**累积重要性方法测试**：

| 配置 | 目标 | 实际 | 特征数 | binary_logloss |
|------|------|------|--------|----------------|
| 测试1 | 95% | 63.16% | 1000（上限） | 0.3506 |
| 测试2 | 90% | 81.97% | 1500（上限） | 0.3908（更差） |

**关键发现**：

1. **特征分数长尾分布**
   - Top特征（HSI_Return_60d）：F-test 87-97
   - 平均分数：11-19
   - 大部分特征分数极低

2. **边际效益递减**
   - 从1000增至1500特征：binary_logloss 反而变差
   - 说明增加低质量特征引入噪声

3. **高覆盖率成本过高**
   - 要达到90%覆盖率需要1500+特征
   - 要达到95%覆盖率可能需要2000-3000特征
   - 训练成本和过拟合风险显著增加

**根本原因分析**：

1. **金融市场预测的固有困难**
   - 有效市场假说：股价主要受随机因素影响
   - 没有少量强预测信号能解释大部分变化
   - 信号高度分散，噪声占比大

2. **特征工程质量问题**
   - 4706个交叉特征可能包含大量冗余
   - 生成交互特征时未做有效过滤
   - 建议减少交叉特征数量至1000-2000

3. **样本量不足**
   - 10只股票，约4400个样本
   - 无法稳定估计近5000个特征的重要性

**结论与建议**：

✅ **固定特征数量仍是最优方案**
- 使用F-test选择固定500个特征
- 避免引入低质量特征
- 训练成本可控，性能稳定

❌ **累积重要性方法不适用于当前场景**
- 特征分数分布过于分散
- 高覆盖率成本过高
- 易引入噪声降低模型性能

⚠️ **特征工程优化方向**
- 减少交叉特征数量
- 提升特征质量而非数量
- 考虑特征去重和相关性过滤
