## 编码规范类

### 路径硬编码问题
- **问题**：硬编码绝对路径 `/data/fortune/...` 导致代码无法在不同环境运行
- **解决**：使用 `os.path.dirname(os.path.abspath(__file__))` 获取脚本目录，基于此构建相对路径
- **正确示例**：
  ```python
  script_dir = os.path.dirname(os.path.abspath(__file__))
  data_dir = os.path.join(script_dir, 'data')
  ```

### 注释中的变更标记
- **问题**：注释中添加"新增"、"修改"等字眼干扰大模型理解
- **解决**：注释只描述功能，变更历史由 git 管理
- **正确做法**：`# 计算筹码分布`
- **错误做法**：`# 计算筹码分布（新增）`

### README 内容规范
- **问题**：README 中添加日期标注如"（2026-03-13新增）"导致过时
- **解决**：README 面向当前用户，直接描述功能，版本历史由 git 管理
- **正确做法**：直接描述功能
- **错误做法**：添加反映更新状态的日期标注

## 模型相关类

### 时间序列数据泄漏警告

> **高准确率不一定意味着好模型，必须严格验证数据泄漏**

在时间序列预测中，高准确率（>65%）通常是数据泄漏的信号：
- 检查数据合并时是否使用了 `ignore_index=True`
- 确保日期索引被保留，数据按时间顺序排列
- 验证时间序列交叉验证是否严格按时间顺序分割
- 对比简单基线（恒定预测、动量策略）的准确率

**参考性范围（经验值）：**
| 等级 | 准确率范围 |
|------|-----------|
| 随机/平衡基线 | ≈50% |
| 常见弱信号 | ≈51–55% |
| 有意义的改进 | ≈55–60% |
| 非常好/罕见 | ≈60–65% |
| 异常高（需怀疑） | >65% |

### CatBoost 1天模型过拟合警告

> **CatBoost 1天模型存在严重过拟合风险，不推荐使用**

**问题描述**：
- 准确率 63.09%（±4.33%）标准偏差过高
- 准确率远高于其他模型的1天准确率（~53%）
- 准确率甚至高于 CatBoost 5天（63.96%）和20天（61.88%），违反一般规律

**根本原因**：
- 样本量差异：1天模型训练样本最多，更容易过拟合
- CatBoost 自动分类特征处理可能在短期预测中过度优化
- 短期波动噪声被模型过度学习

**验证结果**：
- 代码审查通过（没有数据泄漏）
- 时间序列交叉验证正确
- 日期索引保留
- **存在严重过拟合（准确率高 + 标准偏差高）**

**模型可信度评估**：
- CatBoost 20天：高可信度，推荐使用
- CatBoost 5天：中等可信度，谨慎使用
- **CatBoost 1天：低可信度，不推荐**

### 融合模型信号稀释警告

> **融合模型表现不如 CatBoost 单模型，建议优先使用 CatBoost 单模型**

**回测对比**（置信度 0.55，28只股票）：
| 模型 | 年化收益率 | 夏普比率 | 买入信号胜率 | 优秀股票占比 |
|------|-----------|---------|-------------|-------------|
| CatBoost 单模型 | 79.54% | 1.14 | 30.96% | 71% (20/28) |
| 融合模型（加权平均） | 3.16% | 0.32 | 22.75% | 7% (2/28) |

**根本原因：信号稀释问题**
- CatBoost 单模型识别出高质量的强信号
- GBDT/LightGBM 识别出低质量的弱信号（噪声）
- 融合模型将强信号与弱信号混合，导致强信号被稀释

**使用建议**：
- 优先使用 CatBoost 单模型进行预测和回测
- 融合模型仅用于对比研究，不建议用于实际交易

### 深度学习模型失效警告

> **深度学习模型（LSTM、Transformer）表现远不如 CatBoost，不适合用于金融时序预测**

**实验结果**（1天预测，3只股票平均）：
| 模型 | 准确率 | F1分数 | 年化收益率 |
|------|--------|--------|-----------|
| CatBoost | 63.09% | 0.6022 | 79.54% |
| LSTM | 51.79% | **0.0000** | 0.00% |
| Transformer | 51.15% | 0.1303 | 0.00% |

**失败根本原因**：
1. **数据量不足**：单个股票约700个样本，深度学习需要>10,000个样本
2. **信噪比极低**：股价数据信噪比<10%，深度学习容易学习噪声
3. **特征工程不足**：深度学习使用原始价格序列，CatBoost使用500个精选特征
4. **序列依赖假设错误**：股价主要是随机游走（有效市场假说）
5. **过拟合风险高**：参数/样本比>140:1（严重过拟合）

**建议措施**：
- 不推荐使用 LSTM 和 Transformer 用于实际交易
- 继续使用 CatBoost 单模型作为主要预测模型
- 深度学习模型仅用于对比研究和学术探索

## 收益率计算类

### 夏普比率计算错误
- **问题**：使用20天收益率而非年化收益率计算夏普比率
- **解决**：必须使用年化收益率
- **公式**：年化收益率 = 20天收益率 × (252/持有天数)

### 收益率指标定义警告

> **收益率指标定义必须明确，避免报告误导**

**20天持有期收益率（平均收益率）**：
- 定义：每次交易持有20天的收益率
- 用途：与预测周期一致，直观反映短期收益

**年化收益率**：
- 定义：将20天收益率转换为年化收益率
- 计算：年化收益率 = 平均收益率 × (252 / 20) = 平均收益率 × 12.6
- 用途：便于与年化无风险利率比较

**报告要求**：
- 必须同时显示20天收益率和年化收益率
- 明确说明收益率是20天持有期收益率
- 在夏普比率计算中明确使用年化收益率

**典型误导案例**：
- 公用事业股：平均收益率 10.36%，年化收益率 130.54%，夏普比率 -0.0385
  - 错误解读："收益率10.36%很高"（未说明是20天收益率）
  - 正确解读："20天收益率10.36%，年化收益率130.54%，但夏普比率为负，风险极高"

## 关键指标理解类

### 胜率 vs 准确率 vs 正确决策比例

> **评估模型性能时，需要同时关注多个指标，而非单一指标**

| 指标 | 定义 | 计算方式 | 使用场景 |
|------|------|---------|----------|
| **胜率** | 买入后是否盈利 | 盈利交易数 / 买入信号总数 | 评估买入信号质量 |
| **准确率** | 预测方向是否正确 | 预测正确数 / 总交易机会数 | 评估方向预测能力 |
| **正确决策比例** | 决策是否正确 | 正确决策数 / 总决策数 | 综合评估决策质量 |

### 高准确率不等于高收益

**2024年10月案例分析**：
- 准确率：86.21%（优秀）
- 买入信号胜率：23.04%（极差）
- 平均收益率：-1.94%（亏损）

**原因分析**：
- 模型正确识别了大部分下跌机会（高准确率）
- 但买入信号质量很差（低买入信号胜率）
- 熊市环境中，即使有买入信号，也难以盈利

**投资启示**：
1. 不能只看准确率，必须同时关注胜率和收益率
2. 在熊市中，模型的价值在于"避雷"而非"捕猎"
3. 需要根据市场环境动态调整策略

### 胜率与准确率背离问题的解决方案

> **准确率高（50-70%）但买入胜率低（20-55%）时，使用类别权重提升买入信号质量**

**问题描述**（Walk-forward板块验证发现）：
- 消费股：准确率 70.99%，买入胜率 54.81%
- 银行股：准确率 65.40%，买入胜率 51.26%
- 环保股：准确率 47.92%，买入胜率 37.33%
- 模型擅长"避雷"但不擅长"捕猎"

**解决方案对比**：
| 方案 | 平均准确率 | 买入信号准确率 | 买入信号数 | 特点 |
|-----|-----------|---------------|-----------|------|
| A: Baseline | 60.27% | 76.94% | 95 | 无类别权重 |
| **B: Balanced** ⭐ | 58.93% | **77.18%** | 92 | 温和类别权重 |
| C: Balanced + Dynamic | 58.40% | 75.97% | 96 | 类别权重+动态阈值 |

**推荐方案B（Balanced + Fixed 0.55）**：
```python
model = CatBoostModel(
    class_weight='balanced',      # 自动平衡类别权重
    use_dynamic_threshold=False   # 固定阈值0.55
)
```

**关键洞察**：
1. 温和类别权重（balanced）提升买入信号准确率 **+0.24%**
2. 动态阈值在短期测试中效果不明显（市场环境变化不足）
3. 整体准确率略有下降（-1.34%）是预期内的权衡
4. F1分数提升 **+18.5%**（0.4020→0.4766），精确率和召回率更平衡

**业界实践**：
- 类别权重是处理类别不平衡的标准做法
- 温和调整（balanced）不会破坏概率校准性
- 动态阈值更适合长期或高波动市场环境

## 文件上传规范类

### GitHub Actions 报告上传格式
- **问题**：上传过多文件格式（.json/.csv/.md）导致 artifact 体积过大
- **解决**：只上传 `.md` 格式的汇总报告，不上传 `.json`/`.csv` 格式
- **原因**：
  - `.md` 格式人类可读，便于快速查看
  - `.json`/`.csv` 格式主要用于程序处理，占用空间大
  - 汇总报告已包含关键信息，单独报告无需上传
- **正确示例**：
  ```yaml
  - name: 上传分析报告
    uses: actions/upload-artifact@v4
    with:
      name: sector-analysis-report-${{ github.run_number }}
      path: output/sector_performance_analysis_*.md
      retention-days: 90
  ```
- **适用范围**：
  - 板块表现分析（sector-analysis.yml）
  - 牛熊市分析（bull-bear-analysis.yml）
  - 排名分析（ranking-analysis.yml）
  - Walk-forward 验证报告

## 模型评估方法错误

### 简单评估方法存在数据泄漏

> **使用训练数据评估模型会导致结果虚高，必须使用严格的时序验证**

**问题描述**：
之前的 `evaluate_sector_model.py` 脚本存在严重问题：
```python
df = feature_engineer.prepare_data(codes, horizon=horizon)
y_pred = model.catboost_model.predict(X)  # 使用所有数据（包括训练数据）
accuracy = accuracy_score(y, y_pred)  # 结果虚高
```

**问题根源**：
- 使用训练数据来评估模型
- 导致数据泄漏（Data Leakage）
- 评估的是记忆能力，不是泛化能力
- 结果不可信（83-90%买入胜率）

**错误示例**：
- ❌ 银行股板块：报告买入胜率 89.78%（数据泄漏）
- ❌ 交易所板块：报告买入胜率 90.21%（数据泄漏）
- ❌ 半导体股板块：报告买入胜率 83.89%（数据泄漏）

- 稳定性评级：中（良好）

| 3 | 126 | -0.28% | 15.09% | 62.26% | -0.3012 | -36.93% |
| 4 | 114 | 0.21% | 31.91% | 57.45% | 0.2146 | -26.95% |

| 6 | 126 | 5.02% | 36.79% | 63.21% | 0.2686 | -95.06% |

**关键发现**：
- ⚠️ **胜率低于随机水平**：29.32% < 50%（模型过于保守，倾向于预测下跌）

### 业界标准性能基准

根据经验数据：

| 等级 | 准确率范围 | 可信度 |
|------|-----------|-------|
| 随机/平衡基线 | ≈50% | 基准线 |

**Walk-forward 验证结果符合这个基准**，说明是真实性能。

### 正确的评估方法

**Walk-forward 验证**（⭐ 推荐）：
```bash
python3 ml_services/walk_forward_validation.py --model-type catboost --horizon 20 --use-feature-selection
```

**核心特点**：
- 每个fold重新训练模型
- 严格的时序分割（避免数据泄漏）
- 评估真实预测能力
- 业界标准方法

**参数配置**：
- 训练窗口：12个月
- 测试窗口：1个月
- 滚动步长：1个月
- 置信度阈值：0.55

### 避免数据泄漏的最佳实践

**DO（应该做）**：
1. ✅ 使用 Walk-forward 验证
2. ✅ 时间序列分割（TimeSeriesSplit）
3. ✅ 每个fold重新训练模型
4. ✅ 独立的测试集（不参与训练）
5. ✅ 严格的日期顺序分割

**DON'T（不应该做）**：

### 类别权重优化测试（2026-03-18）

> **调整类别权重不能有效解决低胜率问题，保持balanced配置是最佳选择**

**测试背景**：
Walk-forward验证显示所有板块胜率接近或低于50%，尝试通过调整类别权重提高胜率。

**测试三种配置**（以半导体股为测试对象）：

| 配置 | 收益率 | 胜率 | 准确率 | 夏普 | 结论 |
|------|-------|------|-------|------|------|

| 方案B (None) | 0.67% | **50.36%** | 68.80% | 0.116 | 收益下降37% |

**关键发现**：
- ❌ **提高上涨权重（1.5倍）**：准确率提升4.46%，但夏普比率大幅下降61%

- ✅ **Balanced仍是最佳**：夏普比率最高，风险调整收益最优

**经验教训**：
1. **类别权重不是解决低胜率的有效手段**
2. **准确率与夏普比率存在权衡**：提高准确率可能以降低风险调整收益为代价
3. **保持balanced配置**：当前配置已是较优选择
4. **解决低胜率应通过其他手段**：

### Fold 7异常下降分析与改进方案（2026-03-18）

> **新特征在特定市场环境下可能出现过度过滤问题，需要引入市场环境自适应机制**

**问题背景**：
银行股Walk-forward验证中，Fold 7（2025-07-01至2025-07-31）出现异常下降：
- 新特征收益率：-1.16%
- 基准收益率：6.27%
- **下降幅度：-7.43%** ❌

**诊断分析**：

| 指标 | 基准（旧特征） | 新特征 | 变化 | 分析 |
|------|---------------|--------|------|------|
| **收益率** | **6.27%** | **-1.16%** | **-7.43%** | 🔴 严重下降 |
| 买入胜率 | 37.14% | 36.11% | -1.03% | 本身就很低 |

| 买入信号数 | 70 | 72 | +2 | 基本持平 |

**根本原因**：

1. **市场环境异常**：2025年7月为震荡市（ADX < 20），本身难以交易
2. **新特征过度过滤**：
   - Volume_Confirmation要求成交量>1.2倍均量，在缩量反弹时过滤了有效信号
   - False_Breakout检测将正常回调误判为假突破

```python
def detect_market_regime(df):
    """
    市场环境识别 - 基于ADX动态阈值
    参考：QuantInsti Market Regime Detection using HMM
    """
    adx_current = df['ADX'].iloc[-1]
    # 动态阈值：基于60日ADX的30%/70%分位数
    adx_low = df['ADX'].rolling(60, min_periods=30).quantile(0.3).iloc[-1]
    adx_high = df['ADX'].rolling(60, min_periods=30).quantile(0.7).iloc[-1]
    
    if adx_current > adx_high:
        return 'trending'      # 趋势市：严格过滤
    elif adx_current < adx_low:
        return 'ranging'       # 震荡市：放宽过滤
    else:
        return 'normal'        # 正常市：标准过滤

# 应用自适应调整
if market_regime == 'ranging':
    # 震荡市：基于Bookmap建议放宽过滤（防止过度过滤）
    df['Volume_Confirmation'] = (df['Volume_Ratio_7d'] >= 1.0).astype(int)  # 1.2→1.0
    df['False_Breakout_Signal'] = (

    ).astype(int)  # 2点→3点（更严格才触发）
```
