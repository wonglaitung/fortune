本文档记录开发过程中遇到的问题和解决方案，帮助避免重复错误。

## 一、核心警告

### 数据泄漏检测

| 准确率范围 | 判断 | 行动 |
|-----------|------|------|
| 52-55% | 正常 | 略优于随机 |
| 55-60% | 良好 | 业界正常水平 |
| 60-65% | 优秀 | 接近天花板 |
| **>65%** | **可疑** | 检查数据泄漏 |

### 模型可信度

| 模型 | 可信度 | 推荐 |
|------|--------|------|
| **CatBoost 20天** | ⭐⭐⭐⭐⭐ | **推荐** |
| CatBoost 5天 | ⭐⭐⭐ | 谨慎使用 |
| CatBoost 1天 | ⭐ | **不推荐**（过拟合） |
| LSTM/Transformer | ⭐ | **不推荐**（F1≈0） |

### 验证方法

| 方法 | 可信度 | 说明 |
|------|--------|------|
| **Walk-forward** | ✅ 唯一可信 | 每个fold重新训练 |
| 简单评估 | ❌ 不可信 | 结果虚高（83-90%） |

### 样本/特征比例的重要性 ⭐ 新增（2026-04-20）

**问题发现**：
- 原配置：28只股票，2年数据，2426个特征
- 样本/特征比仅 **3.8:1**，远低于推荐值（50:1~100:1）
- 导致模型过拟合，fold间准确率波动大（±5-10%）

**解决方案**：
1. 增加股票数量：28只 → 59只
2. 延长数据期间：2年 → 4年
3. 最终样本/特征比：**15.8:1**（仍低于推荐值，但改善显著）

**效果对比**：
| 指标 | 改动前 | 改动后 | 改善 |
|------|--------|--------|------|
| 有效样本 | ~9,100 | ~38,350 | +321% |
| 样本/特征比 | 3.8:1 | 15.8:1 | +321% |
| 验证准确率 | ~57.5% | 60.57% | +5.3% |
| 准确率波动 | ±5-10% | ±2.34% | 稳定性↑ |
| 变异系数(CV) | ~10% | 3.9% | 显著改善 |

**关键教训**：
1. 样本/特征比 < 10:1 时，模型易过拟合
2. 增加样本量能显著改善模型稳定性（CV降低）
3. 准确率提升幅度有限（+3%），但稳定性大幅提升
4. 要达到50:1推荐比例，还需：
   - 继续增加股票至100+只，或
   - 删减特征至500-800个

**代码注释修正**：
- 原注释："特征数量从2530增至2936"（不准确）
- 修正为："特征数量约2426个（基础892 + 交叉特征1534）"

### 标签阈值参数 ✅ 已验证（2026-04-20）

**背景**：尝试用 `min_return_threshold` 过滤小额波动噪音

**实现方式**：
```python
# 原逻辑：收益 > 0 标记上涨
df['Label'] = (df['Future_Return'] > 0).astype(int)

# 新逻辑：收益 > 阈值 才标记上涨（默认0%）
df['Label'] = (df['Future_Return'] > min_return_threshold).astype(int)
```

**验证结果**（59只股票 + 4年数据，Walk-forward 12 folds）：
| 阈值 | 准确率 | 夏普比率 | 建议 |
|------|--------|---------|------|
| **0%（默认）** | 56.03% | **0.96** | ⭐ 推荐 |
| 0.5% | 59.08% | -0.34 | ⚠️ 不推荐 |

**关键发现**：
1. **阈值0%**：准确率略低，但夏普比率高（0.96），标签分布均衡
2. **阈值0.5%**：准确率略高，但夏普比率负值，标签分布失衡
3. **结论**：默认0%更优，保留参数供后续实盘优化

**待验证**：
- ✅ 已完成：降低阈值至 0%，保留参数供后续优化
- 使用 TRAINING_LIST (59只) + 4年数据重新验证

**教训**：标签阈值调整需要验证标签分布是否均衡

### 特征选择 vs 全量特征 ✅ 已验证（2026-04-20）

**背景**：样本量增加后（28只→59只，2年→4年），重新验证特征选择效果

**验证配置**：
- 股票：59只（TRAINING_STOCKS）
- 数据：4年（1460天）
- 方法：Walk-forward 12 folds
- 模型：CatBoost 20天

**对比结果**：

| 指标 | 全量特征(918) | 特征选择(500) | 差异 | 结论 |
|------|-------------|--------------|------|------|
| 准确率 | **56.03%** | 54.96% | +1.07% | ✅ 全量优 |
| 夏普比率 | **0.96** | 0.83 | +0.13 | ✅ 全量优 |
| 收益率 | **33.66%** | 20.58% | +13.08% | ✅ 全量优 |
| 索提诺比率 | **5.87** | 4.23 | +1.64 | ✅ 全量优 |
| 最大回撤 | -0.54% | **-0.29%** | -0.25% | ✅ 500特征 |

**关键发现**：
1. **全量特征仍然更优**：5项指标中4项胜出
2. 样本/特征比改善后（15.8:1），全量特征优势依然明显
3. CatBoost 内置特征选择机制，无需预处理
4. 特征选择仅在回撤控制上有微小优势

**建议**：
- **继续使用全量特征**（918个）
- 不需要预先进行特征选择
- 当样本/特征比 > 15:1 时，全量特征风险可控

**与之前验证对比**：

| 验证时间 | 样本/特征比 | 全量胜率 | 500胜率 | 结论 |
|---------|------------|---------|---------|------|
| 2026-03-27 | 3.8:1 | ✅ 7/9指标 | 2/9指标 | 全量优 |
| **2026-04-20** | **15.8:1** | ✅ 4/5指标 | 1/5指标 | **全量优** |

**结论一致**：无论样本量大小，全量特征均优于特征选择

## 二、交易策略

### 异常策略（两年数据验证）

| 异常类型 | 策略 | 数据支持 |
|---------|------|---------|
| **IF high** | 🔴 减仓 | 5日收益-3.04%，胜率43% |
| **Z-Score + 当日下跌** | 🟢 抄底 | 5日收益+4.12%，**胜率72%** |

**关键洞察**：
- IF high = 系统性风险 → 减仓
- Z-Score + 当日下跌 = 超跌反弹 → 买入
- 下跌异常稀缺（24.1%），稀缺性增加价值

### 三周期一致预测策略 ⭐

#### 恒生指数

| 模式 | 描述 | 20天准确率 | 策略 |
|------|------|-----------|------|
| **110** | 震荡回调 | **90%** | ⭐ 最优做空 |
| 000 | 一致看跌 | 79.19% | 强烈卖出 |
| 111 | 一致看涨 | 70.31% | 强烈买入 |

**最优策略**：震荡回调(110)做空，胜率90%，收益+4.25%

**市场环境影响**：
- 一致看涨：熊市胜率更高（71.43% vs 65.22%）
- 一致看跌：牛市准确率更高（81.63% vs 78.49%）

#### 个股三周期验证（完整模型）⭐ 更新（2026-04-20）

**验证配置**：56只港股，4年数据，Walk-forward 30 folds，完整模型（545特征）

| 验证项 | 恒生指数（简化） | 56只个股（完整） | 结论 |
|--------|----------------|-----------------|------|
| **过滤律** | 20天 > 1天 | 56.54% > 51.35% | ✅ 成立 |
| **传导律** | 1+5天正确提升 | 62.24% > 56.54% | ✅ 成立 |
| **20天准确率** | 80.73% | 56.54% | 差距 -24.19% |

**完整模型八大模式表现（更新）**：
| 模式 | 样本数 | 20天准确率 | 平均收益 | 建议 |
|------|--------|-----------|---------|------|
| 反弹失败 (0-1-0) | 386 | **66.32%** | -1.65% | ⭐ 最优 |
| 一致看跌 (0-0-0) | 1,012 | 63.14% | -2.54% | 止损 |
| 冲高回落 (1-0-0) | 454 | 62.56% | -2.28% | 获利了结 |
| 下跌中继 (0-0-1) | 454 | 61.45% | +4.86% | 观望 |

**关键发现**：

4. **震荡回调下降**：从恒指90%降至个股48.48%

**模型选择建议**：
- **恒指预测**：使用简化模型（80%+准确率）
- **个股预测**：使用完整模型（56%准确率），但需结合其他指标

## 三、开发规范

### 模型阈值配置

**问题**：文档记录"置信度阈值 0.65"容易误解为预测阈值

**澄清**：
| 用途 | 阈值 | 说明 |
|------|------|------|
| **方向预测** | `0.5` | 概率 > 0.5 预测上涨，≤ 0.5 预测下跌 |
| 置信度分级 | `0.65/0.55` | 用于判断信号强弱，不影响方向 |

**代码示例**：
```python
# 方向判断
pred = 1 if prob > 0.5 else 0

# 置信度分级（仅用于报告展示）
if prob > 0.65 or prob < 0.35:
    confidence = '高'
elif prob > 0.55 or prob < 0.45:
    confidence = '中'
else:
    confidence = '低'
```

**教训**：文档应明确区分"预测阈值"和"置信度阈值"

### 特征缓存机制

**问题**：三周期训练/预测时，特征重复计算3次，每只股票约54s

**解决方案**：添加特征级缓存

**缓存架构**：
```
┌─────────────────────────────────────────────────────────────┐
│                      缓存层级                                │
├─────────────────────────────────────────────────────────────┤
│  原始数据缓存（data/stock_cache/）                           │
│  ├── 股票数据: 7天                                          │
│  └── 恒指数据: 1小时                                        │
├─────────────────────────────────────────────────────────────┤
│  特征缓存（data/feature_cache/）⭐ 新增                      │
│  ├── 缓存键: 股票代码 + 数据最后日期                         │
│  ├── 有效期: 7天                                            │
│  └── 内容: 计算好的特征DataFrame（不含标签）                  │
└─────────────────────────────────────────────────────────────┘
```

**性能对比**：
| 场景 | 耗时 | 加速 |
|------|------|------|
| 无缓存 | 54s | 1x |
| 有缓存 | 0.32s | **170x** |

**使用方法**：
```python
# 默认开启缓存
model.train(codes, horizon=20)  # use_feature_cache=True
model.predict('0005.HK')        # use_feature_cache=True

# 禁用缓存（用于调试）
model.train(codes, horizon=20, use_feature_cache=False)
model.predict('0005.HK', use_feature_cache=False)
```

**关键设计**：
- 缓存不含标签（标签依赖于horizon，不同周期标签不同）
- 三周期训练时，1d/5d/20d 共用同一份特征缓存
- 数据更新后缓存自动失效（缓存键包含数据最后日期）

## 四、常见错误与修复

### 1. 动量分析数据泄漏

**问题**：`future_return` 没有正确 shift

```python
# ❌ 错误
future_return = returns.rolling(5).sum()

# ✅ 正确
future_return = returns.rolling(5).sum().shift(-5)
```

**教训**：简单相关性分析可能存在数据泄漏，应以 Walk-forward 验证为准

### 2. 回撤计算错误（固定持有期策略）

**问题**：时间序列回撤对固定持有期策略是错误的

**修复**：使用批次回撤计算

```python
# ✅ 正确：每个信号日期视为独立投资批次
batches = trades.groupby(trades.index).agg({
    'strategy_return': 'mean'
})
cumulative_values = np.cumsum(batch_returns * batch_weights) + 1.0
```

**结果**：最大回撤从 -84% 修正为 -0.55%

### 3. 夏普比率年化错误

**问题**：假设一年可以做 (252/horizon) 次独立交易

**修复**：使用月度收益年化

```python
# ✅ 正确
annualized_return = monthly_return * 12
annualized_std = batch_std * np.sqrt(12)
```

**结果**：夏普比率从 0.29 修正为 0.97

### 4. Walk-forward 样本量不足

| 训练样本数 | 准确率 | 评估 |
|-----------|--------|------|
| <200 | 38% | 不可信 |
| ≥200 | 55-65% | 稳定 |

**教训**：训练样本不足时结果不可信

### 5. 配置与模型不一致 ⭐ 重要

**问题**：`comprehensive_analysis.py` 使用简化模型数据（80%+），但实际预测使用完整模型（56-65%）

**发现过程**：
1. 文档注释说"来源：docs/THREE_HORIZON_ANALYSIS.md 第10章（完整模型）"
2. 但实际数据（81%+）与完整模型验证结果（64%+）不符
3. 调查发现是简化模型数据

**修复**：统一使用完整模型验证数据

| 模式 | 修改前 | 修改后 |
|------|--------|--------|
| 011 探底回升 | 81.34% | 64.64% |
| 110 震荡回调 | 79.87% | 64.61% |

**教训**：配置数据来源必须与注释一致，避免误导决策

### 6. 流程重复计算

**问题**：`run_comprehensive_analysis.sh` 脚本中，步骤2已生成三周期预测CSV文件，步骤4又重新加载模型预测。

**发现过程**：
1. 运行脚本时发现"加载3个模型"、"进行三周期预测"输出
2. 检查发现步骤2已生成 `ml_trading_model_catboost_predictions_1d.csv`、`5d.csv`、`20d.csv`
3. 步骤4的 `extract_ml_predictions()` 函数又重新调用模型预测

**修复**：
- 新增 `--use-cached-predictions` 参数
- 脚本调用时直接读取已有CSV文件

**设计原则**：
- 多步骤流程中，后续步骤应复用前序步骤的输出
- 提供"使用缓存"选项，兼容单独运行场景

### 7. 交易日 vs 自然日计算 ⭐ 重要

**问题**：`target_date` 使用自然日计算，与模型训练的交易日周期不一致。

**发现过程**：
1. 检查 `ml_trading_model.py` 发现 `timedelta(days=horizon)` 使用自然日
2. 模型训练使用交易日（yfinance 历史数据），但预测评估用自然日
3. 20天预测差异可达9-11天

**影响**：
- 预测评估检查错误日期的价格数据
- 准确率统计可能失真

**修复**：
- 改用 akshare 交易日历（`ak.tool_trade_date_hist_sina()`）
- 8797个交易日，完整到 2026-12-31

**代码对比**：
```python
# ❌ 错误：自然日计算
target_date = date + timedelta(days=horizon)

# ✅ 正确：交易日计算
import akshare as ak
df = ak.tool_trade_date_hist_sina()
trading_dates = set(df['trade_date'].astype(str).tolist())
count = 0
current = date
while count < horizon:
    current += timedelta(days=1)
    if current.strftime('%Y-%m-%d') in trading_dates:
        count += 1
```
