## 一、核心警告

### 1. 性能监控提前评估 Bug ⭐

**问题**：`fetch_price` 容错机制导致提前评估未来到期的预测

**教训**：
- "容错"机制可能引入隐蔽 Bug
- 时间相关逻辑必须有边界检查

**代码**：`ml_services/performance_monitor.py:179-182`

### 2. 大模型 Prompt 数据格式 ⭐

**问题**：大模型混淆不同周期的概率（1天 vs 20天）

**解决**：分离数据用途，大模型只看必要数据

**教训**：Prompt 中的数据格式应尽可能简单、唯一

**代码**：`comprehensive_analysis.py:695-930`

### 3. 数据源一致性问题 ⭐

**问题**：实时价格（分钟数据）和收益率（日线数据）来自不同数据源，导致不一致

**解决**：统一使用腾讯财经 API，失败回退 yfinance

**教训**：价格和收益率必须基于同一数据源

**代码**：`comprehensive_analysis.py:1488-1530`

### 4. 重构代码遗漏功能 ⭐

**问题**：重构时删除了传导模式列

**教训**：重构时必须逐一核对原有功能

## 四、开发规范

### 阈值配置

| 用途 | 阈值 | 说明 |
|------|------|------|
| **方向预测** | `0.5` | 概率 > 0.5 预测上涨 |
| 置信度分级 | `0.65/0.55` | 信号强弱，不影响方向 |

### 数据泄漏防护

```python
# ❌ 错误：使用当日数据
df['Feature'] = df['Close'].pct_change()

# ✅ 正确：使用昨日数据
df['Feature'] = df['Close'].pct_change().shift(1)
```

**高风险特征**：所有 `.rolling()` 计算的特征必须 `.shift(1)`

### 特征缓存

| 项目 | 说明 |
|------|------|
| 位置 | `data/feature_cache/` |
| 有效期 | 7天 |
| 加速 | **170x** |

### 交易日计算

```python
# ❌ 错误：自然日
target_date = date + timedelta(days=horizon)

# ✅ 正确：交易日
df = ak.tool_trade_date_hist_sina()
```

## 五、常见错误

| 错误 | 后果 | 修复 |
|------|------|------|
| 动量分析数据泄漏 | 虚高收益 | `future_return.shift(-5)` |
| 回撤计算错误 | -84% | 使用批次回撤 |
| 夏普比率年化错误 | 0.29 | 月度收益年化 |
| Walk-forward 样本不足 | 38% | 样本 ≥ 200 |
| 配置与模型不一致 | 误导决策 | 统一数据来源 |
| 流程重复计算 | 性能浪费 | 使用缓存 |
| 交易日 vs 自然日 | 评估偏差 | 使用交易日历 |
