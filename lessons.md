## 核心警告

### 数据泄漏检测

| 准确率范围 | 判断 | 行动 |
|-----------|------|------|
| ≈50% | 正常（随机基线） | - |
| 55-60% | 良好 | - |
| 60-65% | 优秀 | - |
| **>65%** | **可能数据泄漏** | 检查特征是否使用当日数据 |

### 模型可信度

| 模型 | 可信度 | 推荐 |
|------|--------|------|
| **CatBoost 20天** | ⭐⭐⭐⭐⭐ | **推荐使用** |
| CatBoost 5天 | ⭐⭐⭐ | 谨慎使用 |
| CatBoost 1天 | ⭐ | **不推荐**（严重过拟合） |
| LSTM/Transformer | ⭐ | **不推荐**（F1≈0） |
| 融合模型 | ⭐ | **不推荐**（信号稀释） |

### 验证方法

| 方法 | 可信度 | 说明 |
|------|--------|------|
| **Walk-forward** | ✅ 唯一可信 | 每个fold重新训练，严格时序分割 |

## 交易策略

### 异常策略（两年数据验证：2024-04-01 至 2026-04-01）

| 异常类型 | 策略 | 数据支持 |
|---------|------|---------|
| **IF high** | 🔴 减仓/止损 | 5日收益-3.04%，胜率43% |
| **Z-Score价格异常+当日下跌** | 🟢 抄底 | 5日收益+4.12%，**胜率72%** |
| Z-Score价格异常+当日上涨 | ⚠️ 观望 | 5日收益+1.96%，胜率54% |

### 关键洞察

1. **IF vs Z-Score 区别**：
   - IF high = 多维异常（系统性风险）→ 减仓
   - Z-Score + 当日下跌 = 超跌反弹（抄底机会）→ 买入

2. **下跌异常稀缺**：
   - 只有24.1%的价格异常是下跌触发
   - 稀缺性增加价值：胜率高达72%

3. **异常是"放大器"**：
   - 异常后波动率增加+79%
   - 需要降低仓位、收紧止损

### 动态仓位调整

| 异常级别 | 仓位乘数 | 阈值调整 | 止损调整 |
|---------|---------|---------|---------|
| 高异常 | 0.55-0.60 | +0.03 | 1.5-2.0×ATR |
| 中异常 | 0.65-0.70 | +0.02 | 2.5×ATR |
| 低异常 | 0.75-0.80 | +0.02 | 3.0×ATR |
| 无异常 | 1.0 | - | 3.5×ATR |

### 加密货币策略（重要！）

**股票策略不适用于加密货币**：

| 策略 | 股票效果 | 加密货币效果 | 建议 |
|------|---------|-------------|------|
| IF high → 减仓 | ✅ 有效 | ❌ 无效 | **不使用** |
| Z-Score抄底 | ✅ 强烈推荐 | ⚠️ 高风险 | **禁止使用** |

**加密货币异常处理**：只作为监控信号，不进行方向性交易。

### 预测置信度策略

| 置信度 | 概率阈值 | 预测方向 | 准确率 | 策略 |
|--------|---------|---------|--------|------|
| **high** | P > 0.6 | up | **84.56%** | 推荐使用 |
| **medium** | 0.5 < P ≤ 0.6 | up | **80.65%** | 推荐使用 |
| low | P ≤ 0.5 | down | 24.11% | **避免使用**（牛市失效） |

### 数据泄漏防护

```python
# ❌ 错误：使用当日数据（数据泄漏）
df['Feature'] = df['Close'].pct_change()

# ✅ 正确：使用昨日数据（无泄漏）
df['Feature'] = df['Close'].pct_change().shift(1)
```

**高风险特征**（必须使用 `.shift(1)`）：
- 所有 `.rolling()` 计算的特征
- BB_Position、Price_Percentile、Intraday_Amplitude
- Price_Ratio_MA5/20/50、Support_120d、Resistance_120d

### Git 规范

| 规范 | 说明 |
|------|------|
| 文件上传 | 只提交 `.md` 格式，不提交 `.json`/`.csv` |
| GitHub Actions | 排程控制在 cron，不在代码中重复判断 |
| 推送冲突 | 使用 `git pull --rebase` 或 `|| true` 容错 |

### 代码一致性

当多个脚本实现相同功能时：

1. **使用相同模块**：提取到公共模块（如 `anomaly_detector`）
2. **统一接口设计**：函数签名、参数命名保持一致
3. **同步更新**：添加新功能时，同时更新所有相关脚本

## 常见错误与修复

### 1. 数据泄漏

**症状**：准确率 > 65%

**修复**：
```python
# 所有技术特征使用 shift(1)
df['Price_Anomaly'] = df['Close'].pct_change().shift(1).abs() > 0.05
```

**验证**：
```python
corr_same = df[feature].corr(df['Returns'])
corr_next = df[feature].corr(df['Returns'].shift(-1))
if abs(corr_same) > abs(corr_next) * 1.5:
    print("⚠️ 可能存在数据泄漏")
```

### 2. 夏普比率计算错误

```python
# ❌ 错误：分子分母不匹配
annualized_return = avg_return * (252 / horizon)
return_std = buy_signals['strategy_return'].std()

# ✅ 正确：统一使用买入信号数据
avg_return = buy_signals['strategy_return'].mean()
annualized_return = avg_return * (252 / horizon)
return_std = buy_signals['strategy_return'].std()
```

### 3. 回撤计算错误（固定持有期策略）

**问题**：Walk-forward 验证显示最大回撤 -84%，明显不合理

**原因分析**：
- `actual_return = Close.pct_change().shift(-20)` = 未来20天的收益率
- 旧代码假设收益在信号当天实现，但实际收益在20天后才实现
- 时间序列回撤计算对于固定持有期策略是错误的

**修复方法**：使用批次回撤计算

```python
# ❌ 错误：按信号日期计算时间序列回撤
trades_by_date = trades.groupby(trades.index)['strategy_return'].mean()
cumulative_values = np.cumsum(daily_returns) + 1.0

# ✅ 正确：每个信号日期视为独立投资批次
batches = trades.groupby(trades.index).agg({
    'strategy_return': 'mean',
    'signal': 'count'
})
batch_returns = batches['strategy_return'].values
batch_weights = np.ones(len(batch_returns)) / len(batch_returns)
cumulative_values = np.cumsum(batch_returns * batch_weights) + 1.0
```

**验证结果**：
| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| 平均最大回撤 | -84.06% ❌ | -0.55% ✅ |

**关键教训**：固定持有期策略的多笔交易是同时持有的，不能使用顺序累乘的回撤计算方法

### 4. 夏普比率年化计算错误（固定持有期策略）

**问题**：夏普比率 0.29 明显偏低

**原因分析**：
```python
# ❌ 错误：假设一年可以做 (252/horizon) 次交易
annualized_return = avg_return * (252 / horizon)  # = 9.42% * 12.6 = 118%
annualized_std = return_std * np.sqrt(252 / horizon)  # 过高
```

**问题本质**：固定持有期策略的多笔交易是**同时持有**的，不能假设一年可以做 12.6 次独立交易

**修复方法**：基于月度收益年化
```python
# ✅ 正确：月度收益 * 12 年化
monthly_return = avg_return  # 月度组合收益
annualized_return = monthly_return * 12

# 年化标准差：批次收益波动 * sqrt(12)
batch_std = np.std(batch_returns, ddof=1)
annualized_std = batch_std * np.sqrt(12)
```

**验证结果**：
| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| 夏普比率 | 0.29 ❌ | 0.97 ✅ |
| 年化收益 | 118% ❌ | 113% ✅ |

**关键教训**：年化因子必须考虑策略的实际交易机制，固定持有期策略应使用月度收益年化

### 5. 数据合理性问题（模型验证指标异常高）

**问题**：平均月收益 9.42%，年化收益 94%，远超业界标准 15-30%

**分析过程**：
1. 检查 `actual_return` 定义：`pct_change().shift(-horizon)` = 未来第horizon天单日收益
2. 尝试使用累积收益 `shift(-horizon)/Close-1` 导致极端值（收益率 259%）
3. 发现模型训练时 `Label` 基于 20天累积收益，验证时使用单日收益

**原因**：
- 累积收益受除权除息、停牌复牌影响，易产生异常值
- `pct_change()` 单日收益数值范围合理（±10%）
- 模型预测的是"方向"，不是具体收益率

**结论**：
- 保持使用 `pct_change()` 进行验证（数值稳定）
- 高收益可能是高置信度阈值(0.65)筛选效应
- 实盘预期收益应打折扣

**关键教训**：
1. 验证指标异常高时，必须检查数据定义是否一致
2. 累积收益计算需要处理异常值（除权除息、停牌）
3. 回测收益应考虑交易成本、滑点等实际因素

### 6. 时区问题

```python
# ❌ 错误：对已有 timezone 的对象使用 tz_localize()
start_date = pd.to_datetime(start_date).tz_localize('UTC')

# ✅ 正确：判断后选择方法
start_date = pd.to_datetime(start_date)
if start_date.tzinfo is not None:
    start_date = start_date.tz_convert('UTC')
else:
    start_date = start_date.tz_localize('UTC')
```
