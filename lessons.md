## 一、核心警告

### 1. 恒指与个股的因果链模式不同 ⭐

**问题**：恒指的 5d→20d 概率传导为正（r=+0.43），个股为负（r=-0.17），完全反向

**教训**：
- 恒指是"均值"，个股噪声被对冲，传导效应清晰
- 个股短期预测概率高反而容易均值回归（反转效应）
- **不能将恒指因果链结论直接套用于个股**

**代码**：`ml_services/analyze_causal_chain.py`、`ml_services/analyze_stock_causal_chain.py`

### 2. 训练窗口不是越大越好 ⭐

**问题**：样本/特征比 7.6x 低于推荐 10x，但增加窗口反而降低准确率

**验证**：12个月 60.11% > 18个月 56.57% > 24个月 54.85%

**教训**：
- 港股近年结构性变化，近期数据比更多历史数据更有预测价值
- Walk-forward 机制保证时效性，弥补样本量不足
- CatBoost 正则化能有效处理高维问题

### 3. 1d 预测有"反向指标"效应 ⭐

**问题**：5个1d预测全错时，5d准确率高达 87.5%（高于全对的 67.4%）

**教训**：
- 1d预测噪声大，全错可能是"反向信号"
- 短期预测的大幅偏离可作为中长期趋势的辅助判断
- 但样本量较小（16次），需谨慎参考

### 4. 性能监控提前评估 Bug ⭐

**问题**：`fetch_price` 容错机制导致提前评估未来到期的预测

**教训**：
- "容错"机制可能引入隐蔽 Bug
- 时间相关逻辑必须有边界检查

**代码**：`ml_services/performance_monitor.py:179-182`

### 2. 大模型 Prompt 数据格式 ⭐

**问题**：大模型混淆不同周期的概率（1天 vs 20天）

**解决**：分离数据用途，大模型只看必要数据

**教训**：Prompt 中的数据格式应尽可能简单、唯一

**代码**：`comprehensive_analysis.py:695-930`

### 3. 数据源一致性问题 ⭐

**问题**：实时价格（分钟数据）和收益率（日线数据）来自不同数据源，导致不一致

**解决**：统一使用腾讯财经 API，失败回退 yfinance

**教训**：价格和收益率必须基于同一数据源

**代码**：`comprehensive_analysis.py:1488-1530`

### 4. 重构代码遗漏功能 ⭐

**问题**：重构时删除了传导模式列

**教训**：重构时必须逐一核对原有功能

## 四、开发规范

### 阈值配置

| 用途 | 阈值 | 说明 |
|------|------|------|
| **方向预测** | `0.5` | 概率 > 0.5 预测上涨 |
| 置信度分级 | `0.65/0.55` | 信号强弱，不影响方向 |

### 数据泄漏防护

```python
# ❌ 错误：使用当日数据
df['Feature'] = df['Close'].pct_change()

# ✅ 正确：使用昨日数据
df['Feature'] = df['Close'].pct_change().shift(1)
```

**高风险特征**：所有 `.rolling()` 计算的特征必须 `.shift(1)`

| 位置 | `data/feature_cache/` |

## 七、分析器设计经验

### 近期指标权重配置

**场景**：风险回报率分析需更关注近期表现

**方案**：添加 `recent_bias` 参数动态调整权重

```bash
# 参数说明
--recent-bias 0.5  # 平衡近期和历史（默认）
--recent-bias 0.7  # 偏向近20日（推荐）
--recent-bias 1.0  # 完全使用近期数据
```

**实现要点**：
- 风险评分：混合近期和历史指标（最大回撤、波动率、Beta）
- 回报评分：提升近期表现权重（10%→17%），降低全周期指标（夏普比率 10%→5%）
- 混合公式：`score = recent * bias + historical * (1 - bias)`

**教训**：
- 不同时间窗口的指标有不同价值，应可配置
- 近期数据更反映当前市场状态，但历史数据提供稳定性
