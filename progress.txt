### 夏普比率计算验证

**问题**：Walk-forward 验证报告夏普比率 4.64，需确认计算是否正确

**验证结论**：计算正确，但需谨慎解读

**计算逻辑验证**：
```python
holding_period_factor = 252 / horizon  # 20天持有期 = 12.6
annualized_return = avg_return * holding_period_factor
annualized_std = batch_std * sqrt(holding_period_factor)
sharpe = (annualized_return - 0.02) / annualized_std
```

**高夏普比率原因分析**：

| Fold | 测试期间 | 正收益批次% | 夏普比率 |
|------|---------|------------|---------|
| 1 | 2025-01 | 100% | 6.51 |
| 2 | 2025-02 | 100% | 7.78 |
| 6 | 2025-06 | 100% | 7.81 |
| 8 | 2025-08 | 100% | 6.77 |
| 12 | 2025-12 | 100% | 10.27 |

**关键发现**：
- **7 个 Fold 的正收益批次比例达到 100%**，导致批次收益标准差很小
- 这反映了 2025 年港股市场的强劲表现，多个月份几乎所有交易日都是正收益
- 夏普比率标准差 4.12，稳定性评级为"低"
- Fold 3 和 Fold 11 表现差（负夏普比率），说明模型在某些市场环境下表现不佳

**建议**：
- 夏普比率 4.64 是正确的，但可能不可持续
- 应关注稳定性指标，而非仅看平均值
- 实盘测试时应设置止损，避免类似 Fold 3 的大幅亏损（-5.97% 回撤）

### 训练时保留基本面特征的 NaN

**问题**：`df.dropna()` 删除所有含 NaN 的行，导致基本面特征缺失时数据丢失

**修改**：
- LightGBM/GBDT/CatBoost 训练时只删除标签和关键列的 NaN
- 保留基本面特征的 NaN，让模型自动处理

**关键列（必须非空）**：Label, Return_1d, Return_5d, Return_20d, Close, Volume

**基本面特征（允许 NaN）**：PE, PB, Market_Cap, ROE, ROA, EPS, Net_Margin, Gross_Margin

**技术原理**：
- LightGBM: `use_missing=True` 自动处理 NaN
- XGBoost/CatBoost: 原生支持 NaN

### Walk-forward 验证收益率计算 Bug 修复

**问题**：`_calculate_metrics` 函数错误地重新计算 `actual_return`，导致收益率数据异常

**现象**：
- 修复前：所有负收益率都在 [-1%, 0%] 区间，正收益率可达 187%
- 修复后：收益率分布正常，范围 [-73%, 125%]

**根本原因**：
- 测试数据被过滤到测试日期范围（如 2025-01-01 到 2025-01-31）
- `actual_return = Close.shift(-horizon) / Close - 1` 在过滤后的数据上计算
- 由于数据只有 19 行，`shift(-20)` 大部分返回 NaN
- 但 `Future_Return` 列在过滤前已正确计算

**解决方案**：
```python
# 优先使用 Future_Return（在过滤前计算）
if 'Future_Return' in df.columns and df['Future_Return'].notna().sum() > 0:
    df['actual_return'] = df['Future_Return']
else:
    df['actual_return'] = df['Close'].shift(-self.horizon) / df['Close'] - 1
```

**验证结果**（CatBoost 20d，12 folds，57只股票）：

| 指标 | 数值 | 评估 |
|------|------|------|
| 综合评分 | 95/100 | 优秀 |
| 平均夏普比率 | 4.43 | ✅ 优秀 |
| 平均 IC | 0.205 | ✅ 有效 |
| 平均 Rank IC | 0.220 | ✅ 有效 |

**高置信度预测错误分析**（prob >= 0.65）：

| 指标 | 值 |
|------|-----|
| 总样本数 | 1,539 |
| 平均损失 | **-6.91%** |
| 中位数损失 | -4.92% |
| 最大损失 | **-72.96%** |
| 损失 <= -5% | **49.4%** |
| 损失 <= -10% | **24.7%** |

**关键发现**：
- **之前的假设（高置信度预测错误损失小）是错误的**
- 近一半的高置信度错误损失超过 5%
- 置信度越高，损失不一定越小（0.95-1.00 区间平均损失 -10.04%）
- 高置信度预测必须配合止损策略

**改动文件**：
- `ml_services/walk_forward_validation.py:422-429`（修复 actual_return 计算）

- 差异：胜率仅计算高置信度交易且扣除交易成本

### 因果链分析：1d→5d 和 5d→20d 的传导关系

**分析脚本**：`ml_services/analyze_causal_chain.py`（恒指）、`ml_services/analyze_stock_causal_chain.py`（个股）

**核心问题**：连续五天1d预测的"积分"能否预测5d方向？连续四次5d预测能否预测20d方向？

**恒指结果**（增强模型，33特征）：

| 指标 | 1d→5d 链 | 5d→20d 链 |
|------|---------|---------|
| 模型直接预测准确率 | 64.14% | **82.76%** |
| 子预测多数投票准确率 | 52.15% | 66.38% |
| 全涨一致准确率 | 63.21% | **89.09%** |
| 全跌一致准确率 | **69.19%** | **89.61%** |
| 传导效应 r | 0.0358 | 0.0811 |
| 概率传导 r | 0.0246 | **0.4294** |

**恒指关键发现**：
- **模型直接预测始终优于子预测积分**（+12% ~ +16%）
- **5d→20d 传导链远强于 1d→5d**（概率传导 r=0.43 vs r=0.02）
- **1d预测有"反向指标"效应**：5个1d全错时5d准确率高达87.5%
- **5d一致信号极强**：全涨87.27%实际上涨率，全跌80.52%实际下跌率

**个股结果**（完整模型，918特征，12只代表股票）：

| 指标 | 恒指 | 个股平均 | 差异 |
|------|------|---------|------|
| 1d→5d 模型准确率 | 64.14% | 51.74% | +12.40% |
| 5d→20d 模型准确率 | 82.76% | 56.59% | **+26.17%** |
| 5d→20d 概率传导 r | **+0.4294** | **-0.1722** | 完全反向 |
| 模型 vs 投票差距(5d→20d) | +16.38% | +12.78% | 一致 |

**恒指 vs 个股关键差异**：
- **一致**：模型直接预测优于子预测积分（两者都成立）
- **不一致**：5d→20d 概率传导完全反向（恒指 r=+0.43，个股 r=-0.17）
- **原因**：恒指是"均值"噪声被对冲，个股受公司特定事件影响有反转效应
- **教训**：个股短期预测概率高反而容易均值回归，不能将恒指因果链结论直接套用于个股

**新增文件**：
- `ml_services/analyze_causal_chain.py`（恒指因果链分析）
- `ml_services/analyze_stock_causal_chain.py`（个股因果链分析，完整模型）
- `docs/DATA_MINING_TIER2_PLAN.md`（深度数据挖掘方案）
- `output/causal_chain_analysis.json`（恒指结果）
- `output/stock_causal_chain_analysis.json`（个股结果）
