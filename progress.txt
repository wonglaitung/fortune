## 最新进展（2026-05-06）

### P12-P15：数据泄漏修复 + 板块相对动能 + QuerySoftMax + Expected_Value ✅

**核心发现**：15+ 个成交量相关特征存在数据泄漏，导致 IC 正但 Rank IC 负

**根因分析**：
- 成交量（Volume）是价格的事后信号，当日成交量可推断当日涨跌
- 模型通过当日成交量"偷看"了当日价格走势，制造 IC 虚高假象
- 实盘预测时无法获得当日收盘成交量，排序逻辑完全失效

**修复内容**：

| 阶段 | 内容 | 文件位置 |
|------|------|---------|
| **P12** | Volume/Turnover 数据泄漏修复 | `ml_trading_model.py:644-687, 822-837` |
| **P13** | 板块相对动能特征 | `ml_trading_model.py:5819-5890` |
| **P14** | QuerySoftMax 损失函数 | `ml_trading_model.py:7527` |
| **P15** | Expected_Value 计算 | `ml_trading_model.py:7959-8058` |

**新增特征**：
- `Sector_Relative_Momentum_5d`：短期板块相对动能
- `Sector_Relative_Momentum_20d`：中期板块相对动能
- `Expected_Value`：`(2*Prob-1) × ATR_Ratio`，胜率转期望收益

**损失函数更新**：
- CatBoostRanker 默认损失函数从 `YetiRank` 改为 `QuerySoftMax`
- QuerySoftMax 对样本权重兼容性更好

**核心教训**：
- **成交量是价格的事后信号**，必须严格使用 shift(1)
- **OBV 是累加指标**，必须整体滞后，否则模型可 100% 推断当日涨跌
- **IC 虚高是数据泄漏的典型症状**，需检查 Rank IC 是否一致

**改动文件**：

### P10 关键发现与修复 - Pairwise 损失函数不支持样本权重

**目标**：修复 P9 Rank IC 变负的根因

**关键发现** 🚨：
- CatBoost 警告：`"Pairwise losses don't support object weights"`
- YetiRank 和 YetiRankPairwise 损失函数**不支持样本权重**
- 代码中传递的 `weight=fold_weights` 参数被**静默忽略**
- 模型对 2019 年陈旧数据和 2024 年最新数据"一视同仁"

**这完美解释了 P9 的异常**：

| 现象 | 原因 |
|------|------|
| IC 达标 (+0.0137) | L1 核心特征有预测力，模型学到"好股票"通用特征 |
| Rank IC 变负 (-0.0087) | 权重失效，模型学到过多"历史过期逻辑"，排序反向 |
| 预测分布挤压 (0.7-1.0) | 模型拟合所有时期（权重相同），产生平均化预测 |

**修复方案**：
- 将 `weight` 改为 `group_weight`（给每个交易日加权）
- YetiRank/YetiRankPairwise 支持 `group_weight` 参数

**代码修改**（`ml_services/ml_trading_model.py:7726-7746, 7769-7785`）：

```python
# P10 修复：计算每个 Group 的权重
group_weights_train = train_df.groupby('group_id')['weight'].first().values

train_pool = Pool(
    data=X_train_fold,
    label=y_train_fold,
    group_id=group_ids_train,
    group_weight=group_weights_train  # ✅ 支持组权重
)
```

**预期效果**：
- 模型优先保证近期交易日排序正确
- Rank IC 由负转正

**改动文件**：
- `ml_services/ml_trading_model.py`（weight → group_weight）
- `docs/FEATURE_LABEL_COMPATIBILITY_PLAN.md`（P10-10~13 章节）

**验证结果**：❌ 无效，见上方 P10 验证结果

## 历史进展（2026-05-04）

### CatBoostRanker 排序模型验证

**目标**：使用 CatBoostRanker + YetiRankPairwise 损失函数优化排序能力，提升 Rank IC

**背景**：
- 当前模型使用 CatBoostClassifier + Logloss 优化分类准确率
- 评估指标 Rank IC 衡量排序能力，两者不对齐
- CatBoostRanker 直接优化排序（YetiRankPairwise 损失）

**实施内容**：

1. **CatBoostRankerModel 类实现**
   - 损失函数：YetiRankPairwise（成对比较式排序）
   - 标签：连续值（Future_Return）而非二元标签
   - group_id：日期（每天一组）
   - eval_metric：NDCG

2. **概率变换修复**
   - 问题：原始分数分布集中，sigmoid 后概率过于集中在 0.5 附近
   - 解决：温度参数归一化 `temperature = score_std`
   - 分类特征处理：正确传递 cat_features 参数

**四版本验证对比（Walk-forward，12 folds，57只股票）**：

| 指标 | CatBoostClassifier | YetiRank | YetiRankPairwise | P5 软标签 | 目标 |
|------|---------------------|----------|------------------|-----------|------|
| IC | -0.0216 | +0.0122 | **+0.0066** | -0.0022 | >0.01 |
| Rank IC | -0.0012 | -0.0060 | **+0.0038** | -0.0063 | >0.02 |
| 准确率 | 59.38% | 49.69% | 49.80% | 48.57% | - |
| 夏普比率 | 0.8291 | 0.7945 | **0.8103** | 0.7146 | >0.8 |
| 索提诺比率 | 3.54 | 5.61 | **5.06** | 3.14 | >5.0 |
| 最大回撤 | -0.23% | -0.22% | -0.23% | -0.21% | <-20% |

**关键发现**：
- ✅ IC 从负变正（-0.0216 → +0.0066）
- ⚠️ Rank IC 改善但仍未达目标（+0.0038，目标 >0.02）
- ✅ YetiRankPairwise 优于 YetiRank（Rank IC +0.0038 vs -0.0060）
- ❌ MinMax 归一化失败（Rank IC 从 +0.0038 降至 +0.0014）
- ❌ **软标签实验失败**（IC 从 +0.0066 降至 -0.0022，Rank IC 从 +0.0038 降至 -0.0063）

**失败尝试**：
- MinMax 归一化：将分数直接映射到 [0,1]，导致 IC/Rank IC 双双下降
- 软标签：将标签从原始收益率改为截面排名百分位（0~1），导致 IC/Rank IC 双双变负
- 结论：原始收益率作为标签仍是最佳选择，自动温度 sigmoid 变换是最佳概率转换策略

**核心教训**：
- 排序模型优化排序而非分类准确率
- YetiRankPairwise 成对比较式排序更适合截面选股场景
- MinMax 归一化不适合 Ranker 分数转概率（信息损失）
- **软标签不适合 YetiRankPairwise**：丢失收益幅度信息后，模型无法区分"大幅跑赢"和"小幅跑赢"
- 温度参数应使用 score_std，而非 1.0/score_std

**最终配置**：
- 损失函数：YetiRankPairwise
- 标签：原始收益率（Future_Return）
- 概率变换：自动温度 sigmoid（temperature = score_std）
- `use_soft_label=False`（已回退）

**改动文件**：
- `ml_services/ml_trading_model.py`（CatBoostRankerModel 类、温度参数、软标签参数）
- `ml_services/walk_forward_validation.py`（Ranker 模型支持、YetiRankPairwise）
- `docs/FEATURE_LABEL_COMPATIBILITY_PLAN.md`（P3/P4/P5 阶段文档）

### LightGBM 和 GBDT 截面特征支持

**目标**：统一所有模型的截面特征计算逻辑，确保与相对标签兼容

**背景**：
- CatBoost 已实现截面百分位和 Z-Score 特征（`_CS_Pct`, `_CS_ZScore`）
- LightGBM 和 GBDT 此前仅支持逐只预测，截面特征退化为 0.5/0.0
- 单只股票预测无法计算截面排名（groupby 返回单一值）

**实施方案**：

1. **批量预测架构改造**（所有模型统一）
   - 阶段1：逐只提取原始特征（不含截面特征）
   - 阶段2：合并所有股票数据，联合计算截面特征
   - 阶段3：逐只预测（使用正确的截面特征值）

2. **新增核心方法**
   - `_extract_raw_features_single()` - 提取单只股票原始特征
   - `_calculate_cross_sectional_percentile_features()` - 计算截面百分位
   - `_calculate_cross_sectional_zscore_features()` - 计算截面 Z-Score
   - `_predict_from_features()` - 从特征数据预测（共用逻辑）
   - `predict_batch()` - 批量预测入口

3. **截面特征覆盖（55个）**
   - 波动率：Volatility_5d/10d/20d/60d/120d, GARCH_Conditional_Vol
   - ATR：ATR, ATR_Ratio, ATR_Risk_Score
   - 成交量：Volume_Ratio_5d/20d, Volume_Volatility, OBV, CMF
   - 动量：Momentum_20d, Momentum_Accel_5d/10d, MACD_histogram
   - RSI：RSI, RSI_Deviation, RSI_ROC
   - 相对强度：RS_Ratio_5d/20d, RS_Diff_5d/20d, Relative_Return
   - 布林带：BB_Position, BB_Width
   - 风险：Max_Drawdown_20d/60d, Vol_Z_Score
   - 资金流向：Smart_Money_Score, Accumulation_Score
   - 基本面：PE, PB, ROE, Market_Cap

4. **单股预测回退机制**
   - 训练时保存 `cs_feature_stats`（各截面特征的均值/标准差）
   - 单股预测缺失截面特征时，使用训练集均值填充

5. **残差化策略改进（P1-5 完成）**

   - **破坏性变更**：旧模型 pkl 文件不兼容，需重新训练

### 数据泄漏分析 + 胜率定义更新

**问题**：训练 CV 准确率（62-71%）高于 Walk-forward 准确率（54.82%），是否有数据泄漏？

**分析结论**：无数据泄漏，属于正常现象

| 验证方法 | 准确率 | 原因 |
|----------|--------|------|
| 训练时 CV | 62-71% | TimeSeriesSplit 数据来自相似时期 |
| Walk-forward | 54.82% | 跨越不同市场环境，更接近真实预测 |

**数据泄漏真实信号**：
- Walk-forward 准确率 > 65%（个股）或 > 80%（恒指）
- 训练 CV 和 Walk-forward 准确率差距 < 5%

**胜率定义更新**：
- 胜率 = 净收益 > 0 的交易数 / 总交易数（扣除双边成本约 0.5%）
- 准确率 = 预测正确数 / 全部样本
- 差异：胜率仅计算高置信度交易且扣除交易成本

### 因果链分析：1d→5d 和 5d→20d 的传导关系

**分析脚本**：`ml_services/analyze_causal_chain.py`（恒指）、`ml_services/analyze_stock_causal_chain.py`（个股）

**核心问题**：连续五天1d预测的"积分"能否预测5d方向？连续四次5d预测能否预测20d方向？

**恒指结果**（增强模型，33特征）：

| 指标 | 1d→5d 链 | 5d→20d 链 |
|------|---------|---------|
| 模型直接预测准确率 | 64.14% | **82.76%** |
| 子预测多数投票准确率 | 52.15% | 66.38% |
| 全涨一致准确率 | 63.21% | **89.09%** |
| 全跌一致准确率 | **69.19%** | **89.61%** |
| 传导效应 r | 0.0358 | 0.0811 |
| 概率传导 r | 0.0246 | **0.4294** |

**恒指关键发现**：
- **模型直接预测始终优于子预测积分**（+12% ~ +16%）
- **5d→20d 传导链远强于 1d→5d**（概率传导 r=0.43 vs r=0.02）
- **1d预测有"反向指标"效应**：5个1d全错时5d准确率高达87.5%
- **5d一致信号极强**：全涨87.27%实际上涨率，全跌80.52%实际下跌率

**个股结果**（完整模型，918特征，12只代表股票）：

| 指标 | 恒指 | 个股平均 | 差异 |
|------|------|---------|------|
| 1d→5d 模型准确率 | 64.14% | 51.74% | +12.40% |
| 5d→20d 模型准确率 | 82.76% | 56.59% | **+26.17%** |
| 5d→20d 概率传导 r | **+0.4294** | **-0.1722** | 完全反向 |
| 模型 vs 投票差距(5d→20d) | +16.38% | +12.78% | 一致 |

**恒指 vs 个股关键差异**：
- **一致**：模型直接预测优于子预测积分（两者都成立）
- **不一致**：5d→20d 概率传导完全反向（恒指 r=+0.43，个股 r=-0.17）
- **原因**：恒指是"均值"噪声被对冲，个股受公司特定事件影响有反转效应
- **教训**：个股短期预测概率高反而容易均值回归，不能将恒指因果链结论直接套用于个股

**新增文件**：
- `ml_services/analyze_causal_chain.py`（恒指因果链分析）
- `ml_services/analyze_stock_causal_chain.py`（个股因果链分析，完整模型）
- `docs/DATA_MINING_TIER2_PLAN.md`（深度数据挖掘方案）
- `output/causal_chain_analysis.json`（恒指结果）
- `output/stock_causal_chain_analysis.json`（个股结果）
