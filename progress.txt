### 2026-05-12 市场情绪过滤器聚合方式修复 ⭐⭐

**问题**：Walk-forward 验证中市场上涨比例计算错误，导致过滤效果与预期不符

**现象**：
- 上午验证 Fold 3 收益 +1.87%，下午验证 -5.66%
- 原因：市场上涨比例计算方式不一致

**根本原因**：
- 上午验证：按日期聚合后计算上涨比例（正确）
- 下午验证：直接使用 Return_1d，未按日期聚合（错误）
- 导致每个股票单独计算上涨比例，而非整体市场上涨比例

**解决方案**：
```python
# 按日期分组计算上涨比例（所有股票中上涨的比例）
daily_up_ratio = returns_df.groupby('Date')['Return_1d'].apply(
    lambda x: (x > 0).mean()
).reset_index()
daily_up_ratio.columns = ['Date', 'Return_1d']
```

**验证效果**：
- 修复后市场上涨比例计算正确
- Fold 3 市场上涨比例 18.6%（极端熊市），正确触发暂停交易

**改动文件**：
- 修改：`ml_services/walk_forward_validation.py`
  - `_apply_market_filter` 方法：按日期聚合后计算上涨比例

### 2026-05-12 CatBoost 分类特征处理一致性修复 ⭐⭐

**问题**：CatBoost 训练时报错 `'data' is numpy array of floating point numerical type, it means no categorical features, but 'cat_features' parameter specifies nonzero number of categorical features`

**根本原因**：
- 分类特征用 LabelEncoder 编码为整数后，numpy 自动将整个数组转为 float 类型
- CatBoost 的 `cat_features` 参数不接受 float 数组

**解决方案**：
- 训练和预测时都不使用 `cat_features` 参数
- 分类特征作为普通数值特征处理
- 保持训练和预测的一致性

**改动文件**：
- 修改：`ml_services/ml_trading_model.py`
  - 训练时移除 `cat_features` 参数
  - `predict_single()` 方法移除 `cat_features`
  - `predict_proba()` 方法移除 `cat_features`

### 2026-05-11 问题Fold根本原因分析与市场环境感知发现

**更新内容**：深入分析问题Fold (3, 10, 11) 的根本原因

**关键发现**：

1. **问题Fold重新定义**：只有 Fold 3 和 Fold 11 是真正的问题Fold
   - Fold 10 盈亏比 1.11，上涨比例 48.9%，表现正常 ✅
   - Fold 3 盈亏比 0.66，上涨比例 18.6%，极端下跌 ❌
   - Fold 11 盈亏比 0.93，上涨比例 26.0%，下跌市场 ❌

2. **高风险股票贡献有限**：
   - 高风险股票（risk >= 85）只贡献 ~20% 的 FP 亏损
   - 过滤高风险股票只能解决小部分问题

3. **核心问题是市场β**：
   | Fold | 上涨比例 | 平均收益 | 市场状态 |
   |------|---------|---------|---------|
   | 3 | **18.6%** | -8.44% | 极端下跌 |
   | 10 | 48.9% | -0.52% | 震荡正常 |
   | 11 | **26.0%** | -3.09% | 下跌市场 |

4. **精细化解决方案**：
   ```python
   if market_up_ratio < 20:      # 极端下跌
       suspend_trading = True     # 暂停交易
   elif market_up_ratio < 30:    # 下跌市场
       threshold = 0.70           # 提高阈值
   elif market_up_ratio < 40:    # 弱震荡
       threshold = 0.65
   else:                          # 正常市场
       threshold = 0.50
   ```

**验证结果**：
- Fold 11 阈值提高到 0.70：盈亏比从 0.93 → **1.09**，净收益从 -823% → **+27%**
- Fold 3 即使阈值提高到 0.70：盈亏比仅 0.77，仍应暂停交易

**结论**：问题本质是"极端β市场未感知"，而非"α预测失效"

### 2026-05-09 特征架构重构与绝对值特征完善

**问题**：市场-网络交叉特征全部使用乘法，未考虑特征单调性方向

**改进方案**：根据特征单调性选择合适的交叉方式

| 交叉类型 | 原方案 | 新方案 | 数学实现 |
|---------|--------|--------|---------|
| 正向 × 正向 | 乘法 | 乘法 | `X × Y` |
| 负向 × 负向 | 乘法 | **风险放大** | `-|X| × |Y|` |
| 正向 × 负向 | 乘法 | **风险调整** | `X / (|Y| + ε)` |
| 涉及中性 | 乘法 | 乘法 | `X × Y` |

**示例**：
- `net_constraint`（结构洞约束）× `VIX`（波动率）= 两者都是负向指标
- 原方案：`0.5 × 20 = 10`（正值，语义混乱）
- 新方案：`-|0.5| × |20| = -10`（负值，风险放大语义清晰）

**改动文件**：`ml_services/ml_trading_model.py`
- 新增 `NETWORK_FEATURE_MONOTONICITY` 字典（7个网络特征）
- 新增 `MARKET_FEATURE_MONOTONICITY` 字典（34个市场特征）
- 新增 `create_monotonic_interaction()` 静态方法
- 修改 `create_market_network_interaction_features()` 使用智能交叉

1. 缓存命中时，代码直接使用缓存的 `stock_df`，跳过交叉特征计算

**解决方案**：
- 将网络特征加载和交叉特征计算移到缓存判断之外
- 无论缓存是否命中，都使用预加载的 `community_ids` 重新计算交叉特征
- 更新缓存以包含最新的交叉特征

**验证结果**：
- 训练时正确预加载社区 ID: `[0, 1, 2, 3, 4, 5, 6]`
- 所有股票生成 434 个市场-网络交叉特征
